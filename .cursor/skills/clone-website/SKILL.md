# Clone Website（参考站复刻）

## Purpose

在**用户拥有合法权利**的前提下，将参考网站拆解为可维护的本地静态或可部署前端：还原信息架构、布局层级、样式与关键交互，并明确资源组织方式。本技能强调合规边界与可访问性、性能，而非绕过站点保护或抓取受保护内容。

## When to Use

- 用户要求「仿站」「复刻」「按某某官网做一版」「clone / replicate / rebuild this site」
- 需要把参考站变成**可编辑代码**（HTML/CSS/JS）与本地预览
- 用户明确持有该站或组件的版权/授权，或仅做**自有品牌**下的「风格对标」且不复制第三方素材

## Do Not Use

- 用户未确认权利归属，或目标为第三方商业站点的完整镜像用于上线
- 需要登录、付费墙、绕过验证码/风控、抓取禁止爬取的数据
- 仅输出截图而无结构化代码交付（除非用户只要静态视觉稿）

## Preconditions（必须先确认）

1. **权利**：站点或设计是否为本人/本公司所有，或是否已获得书面授权；若无，仅允许「原创实现 + 风格借鉴（不复制文案/商标/图片）」。
2. **范围**：单页落地页、营销站若干页，还是含复杂动态路由的全站；是否含后端与账号体系。
3. **目标产物**：纯静态目录、`file://` 可打开，或需接入构建工具（由项目约定）。

## Legal & Ethics（强制）

- **仅**在用户对目标资源拥有权利或明确授权时进行结构/资源级复刻；否则改为「原创页面 + 列出参考点（间距、字号、配色 token）」。
- 遵守 **robots.txt**、站点 **服务条款** 与适用法律；不抓取、不缓存、不再分发受版权保护且未授权的内容。
- **禁止**：伪造身份、窃取凭证、绕过登录、逆向加密接口、批量抓取个人数据。
- 第三方字体/图标/图片：优先替换为自有素材或**明确可商用**授权资源；保留原版权信息字段（如 SIL OFL 字体声明）若适用。

## Scope of Technical Work

| 层级 | 应覆盖 | 说明 |
|------|--------|------|
| 结构 | 语义化 HTML 骨架、主要区块顺序 | 对齐参考的信息架构，避免无意义 div 堆砌 |
| 样式 | 布局（Flex/Grid）、间距、排版、主题色、断点 | 用设计 token（CSS 变量）集中管理，便于改版 |
| 行为 | 导航、折叠、Tab、轮播、表单校验等**可演示**交互 | 无后端时用 mock；不实现未授权的第三方 API 盗用 |
| 资源 | 图片、字体、SVG、音视频路径 | 本地化或 CDN 占位；压缩与合适格式（WebP/AVIF 等按支持度） |
| 非目标 | 服务端逻辑、数据库、原站 CMS 内容同步 | 需单独需求与授权 |

## Workflow

1. **对齐目标**：页面列表、视口（移动/桌面）、必须一致的模块与可简化模块。
2. **结构还原**：自上而下拆区块；为关键区域写简短注释（区块名、用途）。
3. **样式还原**：先版心与栅格，再组件；用相对单位与 `clamp` 等提升响应式；暗色模式若参考站有则按需。
4. **交互还原**：仅实现约定范围内的交互；键盘可操作、焦点可见。
5. **资源处理**：仅使用有权使用的资源；其余用占位图或用户提供的素材包。
6. **本地验证**：静态服务器或构建预览打开全路径，检查控制台无报错、网络无非法外链依赖。
7. **交付说明**：列出与参考站的差异、已知简化项、后续若接 API 的扩展点。

## Tooling Hints for Agents

- **浏览器快照 / 可访问性树**：用于核对 DOM 角色、标题层级、表单标签与焦点顺序；变更布局后重新快照。
- **网络抓取 / Fetch**：仅用于**已授权**且允许获取的公开静态资源；遇 403/登录/反爬即停止并告知用户。
- **本地静态预览**：优先 `npx serve` 或项目已有 dev 命令，避免 `file://` 下 CORS 与模块差异导致误判。
- **对比**：关键屏用同一视口宽度对比间距与字号；记录「刻意不一致」处（品牌、文案、素材）。

## Accessibility

- 语义标签与 `aria-*` 与参考行为一致；图标按钮需 `aria-label`。
- 对比度不低于 WCAG AA（若参考站不足，在交付说明中标注并给出改进）。
- 全键盘可走通主要流程；可见焦点环。

## Performance

- 控制首屏关键 CSS、延迟加载非首屏图片、合适图片尺寸。
- 避免阻塞渲染的大型内联脚本；按需拆分 JS。
- 字体子集化或 `font-display: swap`，减少布局抖动。

## Output Format

- 目录结构清晰（如 `index.html`、`css/`、`js/`、`assets/`）。
- 在 README 或回复中写明：**打开方式**、**主要交互**、**与参考的差异**、**权利与素材来源说明**。
- 若用户仅需技能指引而无代码：输出检查清单与风险提示即可。

## Quality Checklist

- [ ] 已确认授权与范围，非法抓取路径已排除
- [ ] 结构语义化，主要页面与交互可演示
- [ ] 样式 token 化，关键断点已测
- [ ] 资源来源合法，无侵权素材默认入库
- [ ] 可访问性与基础性能已自检
- [ ] 本地预览无控制台错误（在约定浏览器内）
