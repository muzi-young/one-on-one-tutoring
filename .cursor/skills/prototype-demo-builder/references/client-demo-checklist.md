# Client Demo Checklist

Use this checklist before presenting to customers.

## Runtime

- [ ] Install command is documented and works
- [ ] Start command is documented and works
- [ ] Demo opens without runtime errors

## Story coverage

- [ ] 1+ happy path can be completed end-to-end
- [ ] 1+ non-happy path is demoable (empty/error/loading)
- [ ] Navigation between key pages/sections works

## Editability

- [ ] Mock data is centralized (`src/mock/*` or equivalent)
- [ ] Theme or style tokens are centralized
- [ ] Major text copy is easy to locate and change
- [ ] Demo switches/config are in one place

## Visual quality

- [ ] Desktop layout is usable
- [ ] Mobile layout is usable
- [ ] Buttons, inputs, and cards have clear interaction feedback
- [ ] No visible TODO/dummy artifacts

## Presentation readiness

- [ ] A 2-5 minute script is prepared
- [ ] Backup path exists if internet/API fails (local mock path)
- [ ] Known limitations are listed transparently
- [ ] Next iteration items are clear and prioritized
