# Company Intro Generator — 八角科技定制化公司介绍生成器

## 触发与识别

当用户表达以下意图时激活本 skill：
- "帮我给 [行业] 客户写个公司介绍"
- "生成一个针对 [客户类型] 的八角科技介绍"
- "做个 [规模] 企业的合作方案"
- "需要一份定制化的公司简介"

**第一步：识别客户类型**

从用户输入中提取以下信息：
1. **行业类型**（必填）：医疗、电商、本地生活、社交、企业SaaS、物联网、智慧城市、金融科技、新能源、教育、交通等
2. **企业规模**（可选）：大型/上市公司、中型/成长型、初创/创业者、海外/跨国
3. **技术需求**（可选）：高并发、AI、大数据、物联网、区块链、实时通信等
4. **输出格式**（可选，默认Markdown）：Markdown文档 / 飞书云文档 / PPT大纲

如果用户未明确行业，反问确认后再继续。

---

## 素材加载

**必须读取** `references/company-profile.md` — 八角科技完整素材库。

加载后，根据客户类型从素材库中提取对应内容：
- 行业相关能力（"八、客户行业定制素材索引"章节）
- 企业规模定制素材（"九、按企业规模定制素材"章节，如有）
- 技术需求定制素材（"十、按技术需求定制素材"章节，如有）

---

## 生成策略

### 输出结构

生成内容应包含以下模块（按优先级排列）：

1. **开场白**（1-2段）
   - 根据行业定制切入点，说明八角科技与该行业的关联
   - 引用1-2个相关行业案例或数据

2. **公司核心实力**（2-3段）
   - 规模数据：15家分公司、400+研发团队、100000+项目经验
   - 技术能力：与该行业直接相关的技术栈
   - 团队优势：管理团队背景（华为、郑大等）

3. **行业解决方案**（重点，3-4段）
   - 从素材库提取该行业的相关产品/服务
   - 具体功能模块描述
   - 标杆案例（如有）

4. **成功案例**（1-2段）
   - 选取与该行业最接近的案例
   - 数据化呈现成果

5. **服务承诺**（1段）
   - 售后保障：7×24小时、0.1小时响应
   - 开发流程：6步标准化流程

6. **结尾**
   - 邀请进一步沟通
   - 提供联系方式（如需要）

### 内容定制原则

| 原则 | 说明 |
|------|------|
| **相关性优先** | 只提取与客户行业直接相关的能力、案例、产品 |
| **数据说话** | 尽量用具体数字（100000+项目、35%复购率、500+软著等） |
| **案例匹配** | 优先引用同行业案例，次选近似行业 |
| **规模适配** | 大型企业强调标杆案例，中型强调服务响应，初创强调陪伴成长 |
| **避免堆砌** | 不相关的内容不出现，宁可简洁也不冗长 |

## 输出示例

### 示例1：医疗行业客户

用户："帮我给一家医院客户写个公司介绍"

生成逻辑：
- 行业 = 医疗健康
- 提取：智慧医疗、问诊系统、南阳医院信息化项目、仲景医为品牌
- 规模未知，默认中型 → 强调服务响应+医疗行业经验
- 生成约600-800字的定制化介绍

### 示例2：大型电商企业

用户："给一个上市电商公司做合作方案"

生成逻辑：
- 行业 = 电商零售
- 规模 = 大型/上市公司
- 提取：聚爱优选、百丽优选、高并发、商城/社区团购
- 强调：上市公司项目背书、高并发处理能力、全渠道方案

## 输出格式

### 默认：Markdown文档
直接输出格式化Markdown文本，用户可复制使用。

### 飞书云文档（用户要求时）
调用 `feishu-create-doc` skill 或相关工具，将生成的Markdown发布为飞书云文档。

### PPT大纲（用户要求时）
生成PPT页面大纲，每页包含标题和要点，用户可自行制作。

## 限制与注意事项

1. **不做虚构**：所有引用的案例、数据必须来自素材库，不得编造
2. **不泄露敏感**：如素材库未包含某客户具体信息，不擅自推测
3. **不做过度承诺**：避免"保证效果"、"100%成功"等绝对化表述
4. **保持专业**：语气正式但不生硬，适合商务场景
5. **篇幅控制**：默认600-1200字，用户要求长/短时调整
