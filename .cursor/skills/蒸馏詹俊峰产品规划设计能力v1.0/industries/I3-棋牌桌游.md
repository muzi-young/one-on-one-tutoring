# I3 - 棋牌桌游

## L1 领域核心模型

### 核心角色
- **玩家**：休闲或竞技进行传统棋牌、麻将、桌游的用户。
- **开发者/运营商**：开发棋牌大厅、设计赛事、提供防作弊服务。
- **监管机构**：负责棋牌类游戏的版号发放和反赌博监管。

### 核心资源
- **合规资质**：棋牌游戏版号和防沉迷/反赌博系统是产品上架的基础。
- **规则公平性**：稳定的随机发牌算法和防作弊机制。
- **房间/赛事体系**：自由建房、锦标赛、排行榜等社交和组织玩法。

### 核心诉求
- **玩家**：与真人博弈的紧张感和乐趣，打发时间，或赚取虚拟奖励。
- **运营商**：通过房费、道具、赛事报名费变现，控制外挂和赌博行为。
- **平台**：获得用户活跃度和稳定的低风险收入。

### 关键矛盾
- **免费与变相付费**：用户希望纯娱乐，但棋牌天然与“彩头”靠近，涉赌风险高。
- **纯正玩法与房间套路**：有些房间通过机器人控制输赢割韭菜，合规风险极大。
- **用户社交与陌生人匹配**：棋牌适合熟人社交，但线上需陌生人匹配填充房间，体验不易保障。
- **公平性维护与用户作弊**：记牌器、外挂和双开作弊破坏竞技环境，需要持续风控。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：大众棋牌大厅
- **场景描述**：融合斗地主、麻将、中国象棋等多个棋牌品类，一厅多用。
- **关键特性**：通用性强，需适应不同地方规则，运营成本高。

### 形态二：地方特色麻将/扑克
- **场景描述**：只做某个省份或地区的特定玩法牌类。
- **关键特性**：用户精准，粘性极高，地推和社交裂变效果好。

### 形态三：桌游模拟器+社区
- **场景描述**：支持《狼人杀》《剧本杀》等语音社交桌游的线上模拟器。
- **关键特性**：语音交互是核心，剧本内容更新速度决定用户留存。

### 形态四：赛事/金币联赛模式
- **场景描述**：以锦标赛为核心，用户参赛赢积分或奖励。
- **关键特性**：竞技性强，头部赛事观赏性高，可打造主播和电竞赛事。
