# C2 - 消费信贷

## L1 领域核心模型

### 核心角色
- **借款人**：有资金需求、申请信贷额度的个人用户。
- **出借方**：提供信贷资金的金融机构或平台自有资金池。
- **平台运营方**：负责获客、信审、放款管理和催收的技术和运营方。
- **征信机构**：提供个人信用报告和评分的第三方机构。

### 核心资源
- **信贷牌照/资质**：合法从事信贷业务的许可。
- **风控模型**：基于用户数据评估还款能力和意愿的算法模型。
- **资金成本**：平台获取资金的利率决定了信贷产品的价格竞争力。

### 核心诉求
- **借款人**：快速获得资金，额度和利率合理，还款灵活。
- **出借方**：资金安全回收，获得预期利息收益，控制坏账率。
- **平台**：在风险可控的前提下扩大放款规模，赚取利差和手续费。

### 关键矛盾
- **审批速度与风控深度**：秒批秒贷体验好，但可能漏掉欺诈和坏账风险。
- **利率与合规**：高利率覆盖高风险，但超过法定上限可能违规。
- **催收强度与用户体验**：催收太软坏账上升，催收太硬引发投诉和舆论危机。
- **共债风险**：用户在多个平台同时借款，单一平台的风控模型难以完全识别。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：循环额度模式
- **场景描述**：用户获得一个信用额度，随借随还，按日计息。
- **关键特性**：用户使用灵活，资金使用效率高，适合消费场景嵌入。

### 形态二：分期贷款模式
- **场景描述**：用户一次性借款，按期（3/6/12期等）偿还本息。
- **关键特性**：每期还款额固定，用户规划清晰，适合大额消费场景。

### 形态三：先享后付模式（BNPL）
- **场景描述**：用户在消费时选择分期，平台垫付给商家，用户按期还款。
- **关键特性**：与消费场景深度绑定，商家承担部分费用换取转化率提升。

### 形态四：助贷/导流模式
- **场景描述**：平台不承担信用风险，将用户导流给银行等持牌机构。
- **关键特性**：轻资产、合规风险低、收入以引流费为主。
