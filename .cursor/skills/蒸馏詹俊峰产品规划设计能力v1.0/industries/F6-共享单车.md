# F6 - 共享单车

## L1 领域核心模型

### 核心角色
- **骑行者**：短途出行的个人，用单车解决最后一公里。
- **运维人员**：负责车辆调度、维修、更换电池（电单车）和合规摆放。
- **平台运营方**：管理车辆资产、调度秩序和城市关系的技术方。

### 核心资源
- **路面车辆资产**：投放的硬件数量、完好率和位置，是业务的根基。
- **停车和调度能力**：引导用户正确停放、合理调度车辆位置。
- **城市准入和配额**：当地政府允许投放的运营牌照和车数限制。

### 核心诉求
- **骑行者**：随处有车开锁即走，价格便宜，停车限制少。
- **运维人员**：维修任务清晰，取还方便，工资不受投诉影响。
- **平台**：提高每车日均单量，降低损耗和丢失率。

### 关键矛盾
- **车辆堆放与城市管理**：过量投放导致占道和混乱，引起政府限制配额甚至强令退出。
- **自由停放与社会公德**：用户想最大限度就近还车，但规划停放区的设置可能便利性不足。
- **潮汐现象与调度**：早晚高峰地铁口挤满车或完全无车，人工调度成本居高不下。
- **资产折旧与盈利**：单车的丢失率和损坏率使得折旧极快，而定价又要极其便宜。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：无桩共享单车模式
- **场景描述**：无固定停车桩，手机扫码开锁，随处可停。
- **关键特性**：用户体验自由度高，乱停乱放问题突出。

### 形态二：定点还车模式
- **场景描述**：必须骑行到指定停车框或电子围栏内才能还车。
- **关键特性**：城市管理压力小，但部分用户因还车不便而弃用。

### 形态三：共享助力车/电单车
- **场景描述**：带有电瓶和中置电机，骑行更省力，需要用换电方式补充能源。
- **关键特性**：客单价更高，运维更重，但使用范围更广。

### 形态四：景区/校园封闭场景模式
- **场景描述**：只在某个独立区域内投放和运营，离开区域无法还车。
- **关键特性**：偷盗率和丢失率极低，车辆管理简单。
