# B4 - 职场社交

## L1 领域核心模型

### 核心角色
- **职场人士**：建立职业人脉、寻找工作机会、分享行业见解的个人。
- **企业招聘方**：发布职位、搜索候选人、展示雇主品牌的企业 HR 或管理者。
- **内容创作者**：撰写行业分析、职业建议的 KOL 或专家。
- **平台运营方**：维护职业身份的真实性，提供人脉推荐和内容分发。

### 核心资源
- **职业档案**：用户的工作经历、教育背景、技能标签，是最核心的数据资产。
- **人脉图谱**：用户之间的职业关联关系（同事、校友、行业同行等）。
- **职位数据**：企业发布的招聘信息，是平台变现的核心资源之一。

### 核心诉求
- **职场人士**：展示职业形象，拓展有效人脉，发现职业机会，获取行业信息。
- **企业招聘方**：高效找到匹配的候选人，推广公司品牌。
- **平台**：保持用户档案的时效性和真实性，提升用户的活跃度和付费意愿。

### 关键矛盾
- **真实性与包装**：职业档案需要真实，但用户倾向于美化自己，验证成本高。
- **内容深度与用户门槛**：深度行业内容吸引高端用户，但可能让普通用户觉得门槛太高。
- **招聘与社交的平衡**：过多招聘信息变成求职平台，过多社交内容弱化职业属性。
- **隐私与人脉公开**：求职意向可能被当前雇主看到，人脉关系涉及商业竞争敏感信息。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：职业档案+人脉图谱模式
- **场景描述**：以个人档案和人脉关系为核心，辅以职位推荐和行业动态。
- **关键特性**：用户身份真实性强，适合中高端职场人群，变现主要靠招聘服务。

### 形态二：内容社区模式
- **场景描述**：以行业文章、职业问答、经验分享为核心，人脉和招聘为辅助。
- **关键特性**：内容驱动活跃，用户停留时间更长，变现可拓展至教育、咨询等。

### 形态三：招聘优先模式
- **场景描述**：核心功能就是招聘，社交属性为招聘服务的匹配和信任背书。
- **关键特性**：与招聘网站直接竞争，需要大量职位和简历数据支撑。

### 形态四：垂直行业模式
- **场景描述**：只聚焦某一行业（如互联网、金融、医疗），深度满足该行业职场人的需求。
- **关键特性**：用户画像精准，变现效率更高，但天花板受限于行业规模。
