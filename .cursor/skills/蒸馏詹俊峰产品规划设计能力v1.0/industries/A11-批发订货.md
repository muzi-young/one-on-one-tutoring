# A11 - 批发订货

## L1 领域核心模型

### 核心角色
- **采购商**：需要批量采购商品的企业或个人经营者（如零售商、餐馆、小批发商）。
- **供应商/厂家**：生产或大批量持有商品的卖家。
- **平台运营方**：提供货品展示、下单、支付和部分履约支持的中间方。

### 核心资源
- **稳定 SKU 供给**：批发客户需要稳定的货源，频繁缺货会流失。
- **价格优势**：批发价格是采购商选择平台的核心因素。
- **交易习惯**：批发交易订单金额大，赊账、账期、定金等模式常见。

### 核心诉求
- **采购商**：找到一手货源，价格低，质量稳定，能赊账或延期付款。
- **供应商**：接到大订单，及时收回款项，减少坏账风险，提升工厂产能利用率。
- **平台**：展示巨大的交易流水 (GMV)，通过金融服务、广告等增值服务变现。

### 关键矛盾
- **线下关系 vs 线上平台**：批发依赖长期合作关系，线上平台容易引发价格透明化和“跳单”。
- **大宗资金与风控**：交易金额大，对支付安全、信用评价、资金垫付的风险控制要求极高。
- **非标品与大订单**：很多批发商品没有标准 SKU，按需定产，订单条款需要双方协商，难以直接“加入购物车”下单。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：信息黄页模式
- **场景描述**：平台仅展示厂家信息和产品图片，联系方式公开，交易完全线下化。
- **关键特性**：风险最低，变现能力弱，容易被跳过。

### 形态二：线上闭环交易
- **场景描述**：从寻源、询价、签合同、支付到发货，全流程在平台内完成。
- **关键特性**：数据沉淀全，可追溯，但需要强运营和信任机制。

### 形态三：供应链综合服务
- **场景描述**：交易平台之外，深度介入物流、报关、供应链金融等环节。
- **关键特性**：服务收入为主，客户替换成本高，模式重。

### 形态四：分销代发
- **场景描述**：小 B（如微商、主播、社区团长）不需要囤货，在平台选品后一键代发。
- **关键特性**：去中心化，降低了小 B 的启动门槛，但物流体验不可控风险高。
