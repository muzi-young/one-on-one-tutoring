# B3 - 兴趣小组

## L1 领域核心模型

### 核心角色
- **组员**：因共同兴趣聚集在一起的用户。
- **组长/管理员**：创建和管理小组的核心人物，制定组规，维护秩序。
- **内容贡献者**：积极发帖、回复，产出优质内容的活跃用户。
- **平台运营方**：提供小组创建工具、内容分发机制和治理框架。

### 核心资源
- **话题内容**：围绕特定兴趣的讨论、分享和创作。
- **社区文化**：每个小组独特的语言风格、内部梗和行为规范。
- **组长**：有领导力和热情的组长是小组的灵魂，组长的流失会导致小组衰落。

### 核心诉求
- **组员**：找到志同道合的人，获得有价值的信息和情感共鸣。
- **组长**：建立和运营一个有影响力的社区，获得认可和尊重。
- **平台**：提升内容多样性和用户粘性，让用户在自己感兴趣的领域深度沉淀。

### 关键矛盾
- **质量与规模**：小组人多了内容可能水化，老人不满；限制规模又不利于增长。
- **组长独裁与民主**：组长权力过大可能导致组员流失，过于民主可能导致小组方向混乱。
- **封闭与开放**：完全封闭的小组缺乏新鲜血液，完全开放的涌入大量无关内容。
- **商业化与社区纯粹性**：组长接广告赚钱可能引发组员反感，平台商业化也需要平衡。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：自由创建模式
- **场景描述**：任何用户都可以创建小组，平台不干预小组的成立，内容由组长自行管理。
- **关键特性**：生态多样化、优质小组自然涌现、劣质小组自生自灭。

### 形态二：官方运营+用户自发模式
- **场景描述**：平台官方创建一些标杆小组，同时允许用户自主创建。
- **关键特性**：品质可控、运营资源投入大、用户创建的小组获得官方扶持的机制需明确。

### 形态三：付费入圈模式
- **场景描述**：加入小组需要付费（一次性或按月），组长从中获得收入分成。
- **关键特性**：组长有直接收入激励，内容质量通常更高，但限制用户规模增长。

### 形态四：问答+圈子融合模式
- **场景描述**：小组讨论与问答功能结合，优质答案被沉淀为小组知识库。
- **关键特性**：内容沉淀效果好，适合知识型、学习型兴趣小组。
