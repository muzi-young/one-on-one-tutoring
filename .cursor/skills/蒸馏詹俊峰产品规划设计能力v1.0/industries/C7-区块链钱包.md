# C7 - 区块链钱包

## L1 领域核心模型

### 核心角色
- **钱包用户**：持有、转账、接收加密资产的个人或机构。
- **钱包服务商**：提供私钥管理、交易签名、资产展示的技术服务方。
- **矿工/验证者**：负责打包交易、维护区块链网络共识的参与者。
- **监管机构**：关注反洗钱、反恐怖融资和消费者保护的政府部门。

### 核心资源
- **私钥安全**：私钥是加密资产所有权的唯一凭证，私钥安全就是资产安全。
- **多链支持**：支持多条公链和代币标准的能力。
- **用户信任**：去中心化钱包不保管私钥，用户对自己的资产安全负全责，需要用户有高度认知。

### 核心诉求
- **用户**：安全保管资产，方便收发和查看，不丢私钥，不被盗。
- **钱包服务商**：扩大用户规模和资产管理规模，通过增值服务（兑换、质押、借贷）变现。
- **监管机构**：防止加密资产被用于洗钱等非法活动。

### 关键矛盾
- **自托管与非托管**：非托管钱包用户完全掌控私钥，安全要求高但自由度高；托管钱包方便但用户依赖服务商。
- **去中心化与监管**：去中心化钱包不收集用户信息，与监管要求的 KYC 和 AML 存在冲突。
- **安全性高与用户体验差**：最安全的冷钱包操作不便，方便的热钱包安全等级较低。
- **单一链与多链**：专注单链体验好但覆盖资产少，多链钱包支持资产多但维护复杂。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：移动端热钱包
- **场景描述**：用户通过手机 App 管理加密资产，私钥存储在设备本地或云端。
- **关键特性**：体验好，适合日常中小额资产管理和支付。

### 形态二：硬件冷钱包
- **场景描述**：用户通过专用硬件设备管理私钥，设备与网络物理隔离。
- **关键特性**：安全性最高，适合长期大额资产存储，使用体验较重。

### 形态三：交易所托管钱包
- **场景描述**：交易所为用户提供托管钱包，用户不掌控私钥。
- **关键特性**：方便快速交易，与交易所深度绑定，信任成本高。

### 形态四：DApp 浏览器钱包
- **场景描述**：作为浏览器插件或内嵌 SDK，连接去中心化应用。
- **关键特性**：是 Web3 生态的入口，与各类 DApp 的兼容性是关键。
