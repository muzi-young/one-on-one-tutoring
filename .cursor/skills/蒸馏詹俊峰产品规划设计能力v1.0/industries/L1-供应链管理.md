# L1 - 供应链管理

## L1 领域核心模型

### 核心角色
- **采购方**：需要采购原材料或成品的企业，关注价格、质量和交期。
- **供应商**：提供产品或原材料的厂家或经销商，关注订单稳定性和回款速度。
- **仓库/物流方**：负责货物存储和运输的服务商。
- **平台运营方**：提供供应链可视化、协同工具和数据分析的技术方。

### 核心资源
- **供应商网络**：经过审核的合格供应商清单和合作历史。
- **库存数据**：各节点的实时库存水位，是避免断料和积压的基础。
- **订单和履约数据**：采购订单、发货通知、入库单、质检报告等全链路单据。

### 核心诉求
- **采购方**：以合理价格按时获得所需物料，减少缺料停产风险，降低库存资金占用。
- **供应商**：获得稳定的订单和预测，按时回款，减少对账和沟通成本。
- **平台**：提升供应链透明度和协同效率，通过 SaaS 订阅和交易抽佣变现。

### 关键矛盾
- **库存安全与资金效率**：多备库存防断料，但占用资金；少备库存省成本，但风险高。
- **信息共享与商业机密**：供应链协同需要共享预测和库存数据，但企业担心信息泄露给竞争对手。
- **供应商多元化与关系深度**：多元化降低对单一供应商依赖，但深度合作能获得更好的价格和服务。
- **长鞭效应**：终端需求小幅波动，向上游传导时逐级放大，导致库存积压或短缺。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：供应商管理（SRM）模式
- **场景描述**：管理供应商生命周期，包括准入、评估、分类和淘汰。
- **关键特性**：供应商维度的管理工具，适合供应商数量多的大中型企业。

### 形态二：采购协同模式
- **场景描述**：从询价、招投标到下单、对账、开票的采购全流程线上化。
- **关键特性**：流程驱动，减少邮件和 Excel 沟通，提高采购效率。

### 形态三：供应链控制塔模式
- **场景描述**：融合多源数据，提供端到端的供应链可视化和预警。
- **关键特性**：类似指挥中心，从原材料到终端消费者全程可见。

### 形态四：产业带/集采模式
- **场景描述**：聚集同一产业带的中小企业，拼单集采降低采购成本。
- **关键特性**：聚合需求，提升对上游的议价力，适合中小企业。
