# I1 - 休闲超休闲游戏

## L1 领域核心模型

### 核心角色
- **玩家**：利用碎片时间进行简单娱乐和放松的用户。
- **开发者**：独立开发者或小型工作室，以快速开发、低成本试错为主。
- **平台/应用商店**：分发和推荐游戏，提供支付和广告变现基础设施。

### 核心资源
- **核心玩法**：极简、易于理解、即时上手的单一玩法机制（如消除、弹射、堆叠）。
- **用户基数**：极为广泛的轻度游戏人群，不限于传统玩家。
- **广告变现**：激励视频广告、插屏广告是主要收入来源。

### 核心诉求
- **玩家**：迅速获得满足感，消磨无聊时间，不需要复杂的学习或长时间投入。
- **开发者**：低成本开发和测试，快速验证玩法，通过广告和内购获利。
- **平台**：收获大量用户活跃度和广告流量。

### 关键矛盾
- **玩法简单与用户留存**：过于简单导致用户很快玩腻，找不到继续下去的理由。
- **广告变现与游戏体验**：过多的广告打断和压迫会加速用户流失。
- **超高淘汰率与持续创新**：应用商店分发机制让低下载量的休闲游戏面临隐性下架。
- **抄袭与差异化**：核心机制易复制，立项后很快出现大量同类产品争夺原版用户。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：单一机制解压游戏
- **场景描述**：减压、切水果、弹一弹等纯物理互动玩法。
- **关键特性**：心智负担为零，画面和音效讲究爽感，广告极少但观看意愿强。

### 形态二：模拟经营+放置
- **场景描述**：开店、种田、养宠物，离线也有收益的放置经营玩法。
- **关键特性**：长线数值成长，用户会每天定时回来看进度，内购点丰富。

### 形态三：闯关+收集
- **场景描述**：通关后有奖励抽卡或角色皮肤，驱动部分用户重复刷体力。
- **关键特性**：关卡设计需要数值研究，收集系统对留存贡献大。

### 形态四：超休闲对战/io 游戏
- **场景描述**：轻量级对战，每局几分钟，如贪吃蛇大作战等。
- **关键特性**：对战和社交带来病毒式传播，需要服务器支持。
