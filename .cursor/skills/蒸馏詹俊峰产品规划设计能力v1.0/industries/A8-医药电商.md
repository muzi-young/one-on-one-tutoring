# A8 - 医药电商

## L1 领域核心模型

### 核心角色
- **患者/消费者**：购买药品和健康产品的用户。
- **药店/药企**：持有药品经营许可证的商家。
- **平台运营方**：需取得药品网络销售资质，确保合规运营。
- **监管部门**：药监局、市场监管部门，监督药品流通合规。

### 核心资源
- **药品经营许可证**：合法销售药品的资质。
- **处方流转能力**：连接医院/医生、患者、药店的电子处方系统。
- **用药安全保障**：确保药品来源可追溯，效期可控，温控到位。

### 核心诉求
- **消费者**：方便买到需要的药，保证是真药，价格合理，处方药能凭方购买。
- **平台**：合规运营，不触犯药品管理法规，建立用户信任。
- **药企**：拓展销售渠道，维护价格体系，获得用药反馈数据。

### 关键矛盾
- **便捷与安全**：处方药无方销售违法，但审核处方会降低购买便捷性。
- **价格竞争与质量**：低价竞争可能导致劣药混入。
- **信息披露与隐私**：用药记录属于高度敏感的健康隐私。
- **线上与线下利益冲突**：线上药价低冲击线下药店，药企需要平衡渠道。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：B2C 网上药店
- **场景描述**：持证药店在电商平台开设旗舰店，向消费者直接卖药。
- **关键特性**：需要《互联网药品信息服务资格证书》，OTC 可自由售，处方药需方。

### 形态二：O2O 即时送药
- **场景描述**：用户从 App 下单，合作的附近药店接单，骑手送药上门。
- **关键特性**：时效 30-60 分钟，配送范围受限，适合急用药品。

### 形态三：互联网医院问诊开方
- **场景描述**：在线医生视频/图文问诊后开具电子处方，药房凭方发药。
- **关键特性**：形成问诊-开方-售药闭环，合规度最高。

### 形态四：慢病管理药房
- **场景描述**：针对慢病患者，定期配送处方药，提供用药提醒和健康管理。
- **关键特性**：订阅制、复购率高、需要电子处方和医保对接能力。
