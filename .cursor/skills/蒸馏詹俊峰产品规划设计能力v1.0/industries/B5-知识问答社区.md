# B5 - 知识问答社区

## L1 领域核心模型

### 核心角色
- **提问者**：带着具体问题来寻求答案的用户。
- **回答者**：提供解答、分享知识和经验的用户。
- **内容审核/编辑**：维护答案质量，筛选优质内容，处理违规信息。
- **平台运营方**：管理内容生态，激励优质回答，控制低质和有害内容。

### 核心资源
- **优质问答**：高赞回答是平台最核心的内容资产，持续吸引搜索流量。
- **专家/KOL**：有专业背景和表达能力的回答者，是内容质量的保证。
- **社区共识**：点赞/反对机制、折叠垃圾回答、精选优质答案的群众监督体系。

### 核心诉求
- **提问者**：获得准确、有用、可操作的答案，越快越好。
- **回答者**：分享知识获得心理满足、行业声望、或直接的经济收入。
- **平台**：积累高质量内容库，提升用户粘性和搜索引擎可见度，追求内容权威性。

### 关键矛盾
- **开放编辑 vs 恶意修改**：允许共同编辑能让答案更完善，但容易引发恶意修改或观点冲突。
- **知识付费 vs 免费流通**：设置付费墙能提升回答者收入，但可能降低内容流通范围。
- **专家流失 vs 全民参与**：社区成长后早期专家可能因环境变化流失，新用户补充不足导致知识断层。
- **内容权威性与流量驱动**：为了流量扶持争议性话题和短平快内容，长期损害专业权威性。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：免费开放社区
- **场景描述**：所有内容免费访问，靠广告、流量和品牌合作变现。
- **关键特性**：内容覆盖面广，流量大，SEO效果好，依赖规模效应。

### 形态二：知识付费平台
- **场景描述**：回答者提供深度内容需付费查看，平台与创作者分成。
- **关键特性**：内容质量高，作者有直接收入动力，但用户规模和搜索流量受限。

### 形态三：专业认证问答
- **场景描述**：回答者需要认证其专业资质（如医生、律师），答案更具权威性。
- **关键特性**：门槛高但可信度高，适合医疗、法律、金融等需要专业背书的领域。

### 形态四：企业/SaaS 知识库
- **场景描述**：问答能力作为企业内部知识管理工具或SaaS产品的一部分。
- **关键特性**：服务B端客户，注重权限管理和知识沉淀，与公开社区的逻辑不同。
