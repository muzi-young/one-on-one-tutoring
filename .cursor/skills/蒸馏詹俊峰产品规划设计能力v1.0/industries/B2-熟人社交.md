# B2 - 熟人社交

## L1 领域核心模型

### 核心角色
- **普通用户**：与亲友、同事保持联系的个体。
- **群主/管理员**：管理群聊的秩序和成员。
- **平台运营方**：提供稳定、安全的通讯基础设施。

### 核心资源
- **通讯录**：用户的真实社交关系链，是熟人社交的根基。
- **聊天记录**：用户的沟通历史，沉淀了回忆和重要信息。
- **群组**：家庭群、朋友群、工作群等，是高频互动的场所。

### 核心诉求
- **用户**：稳定可靠地收发消息，方便地与亲友保持联系，隐私得到保护。
- **平台**：维持高日活和高留存，通过增值服务（表情、红包、支付）变现。

### 关键矛盾
- **隐私与便利**：通讯录匹配方便加好友，但可能暴露社交关系。
- **通知干扰**：群聊活跃度高时消息轰炸，用户需要免打扰又不愿退群。
- **数据所有权**：聊天记录属于用户还是平台？迁移和备份是用户的刚需。
- **商业化与体验**：广告和信息流内容会干扰纯粹的沟通体验。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：纯通讯模式
- **场景描述**：核心功能就是一対一和群聊，支持文字、语音、图片、视频通话。
- **关键特性**：体验极致轻量，变现手段有限，通常作为生态底座存在。

### 形态二：社交平台模式
- **场景描述**：通讯+朋友圈/动态，用户可以看到好友的生活分享并进行互动。
- **关键特性**：用户停留时长更长，信息流可以承载广告变现。

### 形态三：超级应用模式
- **场景描述**：社交+支付+小程序+生活服务，社交关系链成为其他业务的流量入口。
- **关键特性**：生态壁垒极高、用户替换成本极大、监管关注度高。

### 形态四：办公协同模式
- **场景描述**：熟人社交产品打通办公场景，如企业通讯录、工作群、审批流。
- **关键特性**：模糊生活与工作的边界，提升工作沟通效率但可能增加用户心理负担。
