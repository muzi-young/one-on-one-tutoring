# K3 - 财税法务

## L1 领域核心模型

### 核心角色
- **企业经营者**：需要记账报税、规避法律风险的企业主或高管。
- **财会/法务人员**：处理日常财税工作和法律事务的专业人士。
- **服务机构**：代账公司、律师事务所、会计师事务所等第三方服务商。
- **平台运营方**：通过技术工具连接企业和专业服务，提高服务效率。

### 核心资源
- **专业知识库**：会计准则、税法条文、合同模板、案例库。
- **自动处理能力**：发票 OCR、账目自动归类、税务申报表自动生成。
- **服务商网络**：经过认证的代账会计和律师资源池。

### 核心诉求
- **企业**：以合理成本完成合规的记账报税，遇到法律问题时能快速找到可靠帮助。
- **服务机构**：通过平台获客，提高服务效率，减少沟通和文书成本。
- **平台**：通过 SaaS 工具或撮合服务费变现。

### 关键矛盾
- **自动化与人工判断**：发票识别和分录可自动化，但合理避税和税务筹划仍需人类经验。
- **标准化服务与个性化需求**：不同行业和企业规模的财税需求差异大，套餐难以覆盖。
- **合规风险与平台责任**：平台推荐的服务商或工具出错，导致企业被罚款，责任界定不清。
- **数据安全与数据利用**：企业财税数据高度敏感，平台既要防泄露，又希望大数据分析提供增值服务。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：财税 SaaS 工具模式
- **场景描述**：为企业提供自助式记账、开票和报税软件。
- **关键特性**：成本低，适合小微企业，用户自己操作。

### 形态二：撮合服务平台模式
- **场景描述**：企业在线发布财税或法律需求，服务机构竞标或平台匹配。
- **关键特性**：按需使用，适合专项需求，服务质量高度依赖机构。

### 形态三：托管式服务模式
- **场景描述**：平台雇佣全职会计或律师，为企业提供长期持续服务。
- **关键特性**：质量把控更严，企业省心，但平台运营成本较高。

### 形态四：企业合规培训与内容模式
- **场景描述**：以最新税法、法规解读为核心内容，提供培训、短视频和问答，提升认知。
- **关键特性**：构建用户信任，再引导深度服务或工具使用。
