# L3 - 智慧农业

## L1 领域核心模型

### 核心角色
- **农户/农场主**：种植或养殖的生产者，是智慧农业的最终使用者。
- **农技专家**：提供种植建议、病虫害诊断和农事指导的专业人员。
- **农资供应商**：提供种子、化肥、农药等生产资料的商家。
- **平台运营方**：提供物联网监控、农事管理和产销对接的技术方。

### 核心资源
- **环境数据**：土壤温湿度、光照、气象等传感器实时数据。
- **农事知识库**：作物生长模型、病虫害图谱、施肥配比等专业知识。
- **溯源数据**：从播种到收获的全过程记录，是农产品溢价的核心。

### 核心诉求
- **农户**：减少靠天吃饭的不确定性，提高产量和品质，卖出好价钱。
- **农技专家**：扩大服务半径，高效诊断和指导，获得合理的服务收入。
- **平台**：连接农户、专家和农资，通过硬件销售、SaaS 服务和撮合交易变现。

### 关键矛盾
- **技术门槛与使用习惯**：农户年龄偏大、文化程度有限，复杂的数字工具推广困难。
- **投入成本与短期回报**：物联网设备和服务费是成本，农产品价格波动让回报不确定。
- **数据准确性与环境复杂性**：户外传感器受天气、动物等干扰，数据精度不如工厂环境。
- **个体经营与规模化服务**：以家庭为单位的碎片化生产，与标准化技术服务之间存在落差。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：大田种植监控模式
- **场景描述**：在田间部署气象站和土壤传感器，配合无人机巡田，数据汇总分析。
- **关键特性**：适合大规模种植，投入产出比较明显。

### 形态二：设施农业/大棚控制模式
- **场景描述**：智能温室中控系统，自动控制灌溉、遮阳、通风等设备。
- **关键特性**：自动化程度高，环境可控，适合高附加值作物。

### 形态三：养殖管理/智慧畜牧模式
- **场景描述**：通过耳标、项圈等可穿戴设备监测动物健康、位置和发情状态。
- **关键特性**：减少人工巡查，精准饲喂，适合规模化养殖场。

### 形态四：农产品溯源+电商模式
- **场景描述**：以溯源数据为信任基础，连接消费者与产地，实现优质优价。
- **关键特性**：打通产销两端，让种植端的投入通过消费端溢价回收。
