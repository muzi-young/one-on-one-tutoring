# C1 - 移动支付

## L1 领域核心模型

### 核心角色
- **消费者**：使用移动支付进行消费、转账、缴费的个人用户。
- **商户**：接受移动支付作为收款方式的商家。
- **支付平台**：提供支付通道、账户体系和资金托管的技术服务方。
- **银行/清算机构**：提供底层账户和资金清算服务的金融机构。
- **监管机构**：监督支付合规、反洗钱和资金安全的政府部门。

### 核心资源
- **支付牌照**：合法从事支付业务的行政许可，是核心壁垒。
- **账户体系**：用户的虚拟账户，承载余额、积分、优惠券等。
- **支付通道**：连接银行、卡组织、第三方支付的清算网络。

### 核心诉求
- **消费者**：支付快捷方便，资金安全有保障，手续费低或免费。
- **商户**：收款到账快，费率合理，对账清晰，支持主流支付方式。
- **平台**：扩大用户规模和交易量，通过支付数据衍生金融服务变现。

### 关键矛盾
- **便捷与安全**：越便捷的支付方式（如免密支付）资金风险越高。
- **创新与监管**：支付创新加速可能触及监管红线，需要平衡。
- **费率与商户接受度**：支付费率是平台收入来源，但过高会让商户抵触。
- **数据隐私与商业价值**：支付数据极具商业价值，但用户隐私保护要求日益严格。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：电子钱包模式
- **场景描述**：用户在平台内开立虚拟账户，充值后在生态内消费。
- **关键特性**：用户粘性高、资金沉淀大、跨场景使用取决于生态广度。

### 形态二：聚合支付模式
- **场景描述**：平台聚合多种支付方式（银行卡、扫码、NFC），商户接入一个接口即可。
- **关键特性**：商户拓展快、费率透明要求高、与各支付通道的博弈复杂。

### 形态三：扫码支付模式
- **场景描述**：以二维码为载体，消费者扫码付款或商户扫码收款。
- **关键特性**：硬件成本几乎为零，普及速度快，适合线下小额高频场景。

### 形态四：生物识别支付模式
- **场景描述**：通过人脸、指纹、掌纹等生物特征完成支付。
- **关键特性**：体验无感、对识别精度和活体检测要求极高、隐私争议大。
