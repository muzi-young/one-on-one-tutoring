# D6 - 孕期管理

## L1 领域核心模型

### 核心角色
- **孕妇**：孕期中的准妈妈，需要孕期知识、产检提醒和身体管理。
- **孕妇家人（准爸爸等）**：参与孕期管理，获取同步信息。
- **平台运营方**：提供孕期阶段内容、工具和社区，对接母婴消费。

### 核心资源
- **孕期阶段内容**：按孕周推送的宝宝发育知识、孕妇身体变化和注意事项。
- **产检时间表**：标准化的产检项目和提醒。
- **母婴消费入口**：孕期和产后的大量消费需求，是重要的变现通路。

### 核心诉求
- **孕妇**：了解宝宝发育状态，按时完成产检，缓解孕期焦虑，准备待产。
- **平台**：建立用户粘性，从孕期延伸到产后育儿管理，通过电商和广告变现。

### 关键矛盾
- **周期性与留存**：孕期只有约 40 周，用户完成生育后可能离开，需要延伸到育儿。
- **焦虑与信息过载**：孕妇情绪敏感，过多或不准确的信息可能增加焦虑。
- **社区质量与用户量**：孕期社区容易出现焦虑传染、偏方流传，社区氛围管理难。
- **变现与用户体验**：孕期强推商品可能引起反感，需要把握分寸。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：孕期阶段管理工具模式
- **场景描述**：以孕周为轴，推送每日知识、产检提醒和身体记录工具。
- **关键特性**：工具属性强，用户每日使用频次高。

### 形态二：孕期+育儿全周期模式
- **场景描述**：从备孕到孕期、到产后育儿，覆盖完整生命周期。
- **关键特性**：用户生命周期延长，数据积累更有价值。

### 形态三：孕期社群模式
- **场景描述**：以同月预产期、同城、同兴趣等维度组建社群。
- **关键特性**：用户归属感强，社区氛围要求高，需要运营投入。

### 形态四：智能硬件联动模式
- **场景描述**：与胎心仪、体重秤、血压计等家用硬件联动，自动记录数据。
- **关键特性**：数据更客观连续，用户粘性更强，但硬件销售和对接成本高。
