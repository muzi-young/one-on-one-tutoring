# H1 - 短视频

## L1 领域核心模型

### 核心角色
- **内容消费者**：观看短视频、进行互动（点赞、评论、分享）的用户。
- **内容创作者**：拍摄、剪辑并发布短视频的个人或团队。
- **平台运营方**：提供拍摄工具、推荐算法和流量变现机制。

### 核心资源
- **内容供给**：源源不断的创意短视频，是留住用户的核心。
- **推荐算法**：决定用户看到什么内容，直接影响用户时长和留存。
- **创作者生态**：从素人到头部达人的成长和变现通道。

### 核心诉求
- **消费者**：看到有趣有料的内容，杀时间，获得放松或信息。
- **创作者**：获得流量和粉丝，通过广告、打赏、带货等方式赚取收入。
- **平台**：最大化用户使用时长和活跃度，通过广告和电商变现。

### 关键矛盾
- **内容质量与数量**：大量低质内容能填充时长，但会劣化体验导致优质用户流失。
- **算法推荐与信息茧房**：算法迎合短期兴趣，可能导致用户视野变窄。
- **创作者变现与内容客观性**：商业合作内容可能影响创作者的真实表达。
- **版权与二次创作**：大量视频使用未经授权的音乐或影视素材，版权纠纷不断。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：沉浸式单列信息流模式
- **场景描述**：全屏上下滑动，自动播放，内容消费效率极高。
- **关键特性**：适合碎片化时间，用户容易进入沉浸状态。

### 形态二：双列瀑布流发现模式
- **场景描述**：封面多图排列，用户主动点击感兴趣的视频观看。
- **关键特性**：给用户更多选择权，适合内容导向和社区氛围。

### 形态三：直播+短视频融合模式
- **场景描述**：短视频引流，直播变现，两者无缝切换。
- **关键特性**：粉丝经济强，主播与观众实时互动增强粘性。

### 形态四：本地/同城短视频模式
- **场景描述**：优先展示同城内容，强化本地社交和探店消费。
- **关键特性**：与本地生活服务结合紧密，变现路径更短。
