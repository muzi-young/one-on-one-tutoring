# G7 - 记账理财

## L1 领域核心模型

### 核心角色
- **记录者**：希望通过记账控制支出、管理预算、实现储蓄目标的个人或家庭。
- **平台运营方**：提供便捷的记账工具和支出分析。

### 核心资源
- **收支数据**：用户的日常收支记录是最核心的数据资产。
- **自动化记账**：同步银行卡、信用卡、支付宝微信账单的能力。

### 核心诉求
- **用户**：轻松记账不费时，看清钱花在哪里，达成省钱目标。
- **平台**：扩大用户基数和数据积累，通过理财推荐、贷款导流和会员变现。

### 关键矛盾
- **记账繁琐与坚持**：手工记账太累放弃率高，自动化同步又涉及隐私和银行授权。
- **支出控制与消费欲望**：用户想省钱，但平台推的贷款和分期又在鼓励消费。
- **工具属性与变现诉求**：记账工具本身很难收费，靠金融产品导流可能损伤信任。
- **隐私与数据价值**：详尽的个人收支是高价值金融数据，但用户对隐私高度敏感。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：纯记账工具模式
- **场景描述**：手动或同步账单，分类统计，生成消费图表。
- **关键特性**：定位纯粹，收入靠会员和去广告订阅。

### 形态二：记账+预算+多人共享
- **场景描述**：家庭或情侣共享账本，协同管理家庭开支。
- **关键特性**：多用户协作，提升了打开频率和传播性。

### 形态三：记账+智能投顾
- **场景描述**：根据消费和储蓄数据，提供基金定投和理财建议。
- **关键特性**：用数据闭环直接推荐金融产品，变现路径短但合规风险高。

### 形态四：银行/信用卡 App 内置记账
- **场景描述**：由银行提供的记账模块，数据自动归集但只分析本行数据。
- **关键特性**：用户数据完整度受限，但自动化程度最高。
