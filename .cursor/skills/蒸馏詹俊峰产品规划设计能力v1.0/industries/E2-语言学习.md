# E2 - 语言学习

## L1 领域核心模型

### 核心角色
- **学习者**：希望掌握一门新语言的个人，从零基础到进阶不同阶段。
- **教师/母语者**：提供语言教学或口语练习的专业或非专业人员。
- **平台运营方**：提供课程内容、练习工具和学习进度跟踪。

### 核心资源
- **学习内容**：词汇、语法、听力、口语等模块化的学习材料。
- **反馈机制**：发音评测、语法纠错等即时反馈能力。
- **学习激励**：打卡、排行榜、等级等维持学习动力的机制。

### 核心诉求
- **学习者**：高效、有趣地掌握语言，看到自己的进步，能实际应用。
- **平台**：提升用户留存和学习时长，通过课程付费、会员订阅等变现。

### 关键矛盾
- **碎片化与系统性**：用户每天只有碎片时间，但语言需要系统性学习。
- **输入与输出**：听读输入容易提供，口语写作输出需要真人互动。
- **兴趣与坚持**：初始兴趣过后，坚持学习需要强激励。
- **自然习得与系统语法**：沉浸式自然习得体验好，但成年人需要语法框架辅助。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：游戏化打卡学习模式
- **场景描述**：学习内容拆分为每日关卡，用户通关解锁下一级。
- **关键特性**：趣味性强，适合入门和保持习惯，深度有限。

### 形态二：真人对话练习模式
- **场景描述**：连接学习者与母语者，通过视频或语音进行对话练习。
- **关键特性**：口语提升快，匹配和调度成本高，需要管理对话质量。

### 形态三：AI 口语陪练模式
- **场景描述**：语音识别和 AI 对话引擎，学习者与 AI 进行口语练习。
- **关键特性**：随时随地练习，无社交压力，适合初级阶段。

### 形态四：沉浸式情景学习模式
- **场景描述**：通过视频、音频、故事等还原真实语言场景，让学习者在情境中习得。
- **关键特性**：用户体验好，内容生产成本高，适合中等水平学习者。
