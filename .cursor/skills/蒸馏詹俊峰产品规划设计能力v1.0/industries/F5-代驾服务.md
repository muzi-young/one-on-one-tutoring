# F5 - 代驾服务

## L1 领域核心模型

### 核心角色
- **车主/用户**：饮酒后、疲劳或不便驾驶、需要代驾服务的车辆拥有者。
- **代驾司机**：提供驾驶服务的兼职或全职司机。
- **平台运营方**：匹配代驾需求与就近司机，提供安全和服务保障。

### 核心资源
- **司机供给**：夜间运力充足的代驾司机数量，直接决定用户叫到服务的概率。
- **保险和事故处理**：为潜在交通事故提供法律和保险保障，是车主信任的基础。

### 核心诉求
- **车主**：快速找到附近司机，安全到家，收费透明，车辆无损。
- **代驾司机**：稳定收入，合理派单，自身安全有保障，平台抽成合理。
- **平台**：最大化订单匹配效率，严格控制司机质量，避免恶性事故。

### 关键矛盾
- **夜间高峰与司机出勤意愿**：多数需求在深夜，愿意在凌晨路面上工作的司机天然稀缺。
- **事故责任与理赔压力**：代驾期间发生交通事故的定责和赔偿流程复杂，容易产生纠纷。
- **信任与核身**：车主不放心把豪车和自身人身安全交给陌生人，必须强化司机的背景核查。
- **收入保证与防止疲劳驾驶**：司机想多接单赚钱，但疲劳驾驶风险极大，平台容易被追责。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：单次全权代驾模式
- **场景描述**：车主呼叫代驾，司机将车主和车一起送回家后任务完成。
- **关键特性**：场景明确，高频需求集中在晚餐和夜生活后。

### 形态二：预约/长期代驾模式
- **场景描述**：商务活动、接送机、孕妇产检等提前数小时甚至数日预约。
- **关键特性**：客单价更高，用户对时间准点率敏感，司机变动会严重影响满意度。

### 形态三：代保养/代年检模式
- **场景描述**：不只是送人，而是帮车主完成洗车、保养、年检等事务。
- **关键特性**：服务链条更长，对司机的额外技能和责任心有要求。

### 形态四：商务酒后代驾月结模式
- **场景描述**：合作餐厅、酒店、会所，为顾客提供代驾服务，企业按月结算。
- **关键特性**：B2B为主，流量稳定，需要系统对接企业签单和核销。
