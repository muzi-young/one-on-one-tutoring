# F3 - 地图导航

## L1 领域核心模型

### 核心角色
- **出行用户**：需要路线指引和实时路况的驾驶员或行人。
- **开发者/商户**：调用地图 SDK 或 API，在自己的应用中嵌入地图和位置服务。
- **平台运营方**：采集和更新地图数据，提供导航引擎和位置服务。

### 核心资源
- **地图数据**：道路、POI（兴趣点）、实时路况，是地图服务的底层基础。
- **定位能力**：GPS、基站、WiFi 等多源融合的高精度定位。
- **导航算法**：路线规划、ETA 预估、避开拥堵的算法引擎。

### 核心诉求
- **出行用户**：找到正确路线，避开拥堵，预估达到时间准确，操作安全简便。
- **开发者/商户**：快速集成地图能力，调用成本可控，SDK 稳定易用。
- **平台**：扩大用户基数和数据覆盖，通过广告、API调用费和增值服务变现。

### 关键矛盾
- **时效性与成本**：地图数据需要持续更新，采集车和卫星的成本极高。
- **隐私与价值**：用户位置数据是精确推荐和广告的核心，但是否涉及隐私侵犯存争议。
- **精度与覆盖范围**：一线城市可以达到厘米级高精地图，但偏远地区和地下停车场精度骤降。
- **开放平台与自闭环**：开放API给开发者能带来生态价值，但也可能将核心导航流量拱手让人。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：C 端导航应用模式
- **场景描述**：面向个人用户的手机导航 App，提供驾车、步行、骑行导航。
- **关键特性**：用户基数大，数据采集众包化，广告和本地生活服务变现。

### 形态二：车载导航模式
- **场景描述**：预装在汽车车机中的导航系统，与车辆硬件深度整合。
- **关键特性**：屏幕更大，与车辆传感器联动，更新频率不如手机端。

### 形态三：地图 API 平台模式
- **场景描述**：向开发者和企业提供地图 SDK 和 API 服务。
- **关键特性**：B2B 收入为主，技术壁垒高，是生态的基础设施。

### 形态四：高精地图/自动驾驶模式
- **场景描述**：为自动驾驶车辆提供厘米级高精度地图。
- **关键特性**：技术门槛最高，与车企深度绑定，未来前景大但短期内商业化困难。
