# D8 - 体检预约

## L1 领域核心模型

### 核心角色
- **体检者**：预约体检的个人用户，或团检企业中的员工。
- **体检中心/医院**：提供体检服务的机构。
- **企业 HR/行政**：为员工购买和管理团检的企业代表。
- **平台运营方**：提供套餐比较、在线预约和报告查看的技术方。

### 核心资源
- **体检机构合作关系**：合作的体检中心网络覆盖范围。
- **套餐设计能力**：根据人群特征（年龄、性别、职业）设计的差异化套餐。
- **报告解读能力**：体检后提供通俗易懂的报告解读和健康建议。

### 核心诉求
- **体检者**：方便预约，获得准确的检查结果，了解自己的健康状况。
- **体检中心**：获得客源，提高设备利用率，降低空检率。
- **平台**：撮合交易，通过佣金和增值服务变现。

### 关键矛盾
- **标准化与个性化**：体检套餐需要标准化以提高效率，但用户需要个性化的检查项目。
- **价格透明与利润空间**：体检价格越来越透明，平台的抽佣空间被压缩。
- **预约与爽约**：预约制方便用户，但爽约导致医疗资源浪费。
- **报告理解门槛**：体检报告术语专业，用户看不懂需要解读服务。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：体检套餐电商模式
- **场景描述**：平台展示不同机构的体检套餐，用户比较和下单购买。
- **关键特性**：交易属性强，价格敏感用户多，需要丰富的 SKU。

### 形态二：定制化体检模式
- **场景描述**：用户完成健康问卷，AI 推荐专属体检项目组合。
- **关键特性**：用户价值感和匹配度更高，但客单价更高，决策成本更大。

### 形态三：企业团检管理模式
- **场景描述**：企业 HR 通过平台采购团检，员工自主预约，平台统计使用情况。
- **关键特性**：B2B2C 属性，获客成本高但订单金额大，续约是关键。

### 形态四：检后健康管理模式
- **场景描述**：体检不是终点，平台根据体检结果推荐生活方式改善或复查建议。
- **关键特性**：服务链条更长，用户粘性更强，需要对接慢病管理或在线问诊。
