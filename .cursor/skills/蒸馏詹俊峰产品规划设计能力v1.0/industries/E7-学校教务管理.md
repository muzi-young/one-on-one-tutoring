# E7 - 学校教务管理

## L1 领域核心模型

### 核心角色
- **教务老师/管理者**：排课、管理学籍、统计成绩的核心操作者。
- **教师**：录入平时成绩和期末成绩，使用课表和考勤。
- **学生/家长**：查课表、看作业、看成绩通知。
- **区域教育局/领导**：查看区域学校运行数据和报表。

### 核心资源
- **组织架构树**：学校-年级-班级-学科，精细的层级关系是教务的基础。
- **学籍和成绩数据**：最核心也最敏感的学校数据。
- **决策报表**：为学校管理和区域治理提供的宏观数据支持。

### 核心诉求
- **管理者**：快速排课无冲突，随时调取学籍和成绩，期末考情分析自动生成。
- **教师**：考勤和成绩录入方便，不花太多额外时间。
- **学生/家长**：不错过上课时间，及时收到学校通知，清晰了解整体学业。
- **平台**：提高学校管理效率，通过增值服务变现，但前提是不增加老师负担。

### 关键矛盾
- **管理需求与一线使用意愿**：校领导需要数字化管理，但要一线老师额外录入数据就很难推动。
- **标准化产品与学校个性化差异**：每个学校有自己的管理习惯，标准软件难以满足所有定制需求。
- **数据安全与互联互通**：教育信息敏感，同时需要和区域系统、招生系统等外部系统对接。
- **预算有限与持续服务**：学校购买力一次性多，持续性少，导致服务商必须不断拓展新客户。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：单体学校 SaaS 模式
- **场景描述**：云端的课表、成绩和通知系统，按年付费。
- **关键特性**：部署快，维护成本低，适合中小学和培训机构。

### 形态二：区域教育云平台模式
- **场景描述**：以区县教育局为采购方，辖区所有学校统一使用。
- **关键特性**：自上而下推动，覆盖广，项目制为主，政策导向性强。

### 形态三：家校互通工具模式
- **场景描述**：聚焦考勤、作业和通知发布，老师和家长为主要用户。
- **关键特性**：高频使用，班级群和班级圈粘性高，可渗透电商和内容。

### 形态四：走班排课专业工具
- **场景描述**：配合新高考走班制教学，解决分科走班带来的复杂排课问题。
- **关键特性**：排课算法是核心竞争力，售后支持需要熟悉新高考政策。
