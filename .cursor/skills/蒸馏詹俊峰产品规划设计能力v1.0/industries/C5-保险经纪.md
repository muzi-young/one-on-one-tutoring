# C5 - 保险经纪

## L1 领域核心模型

### 核心角色
- **投保人**：购买保险产品以转移风险的个人或企业。
- **保险经纪人**：代表投保人利益，帮助比较和选择保险产品的中介。
- **保险公司**：承担风险并提供理赔服务的持牌机构。
- **平台运营方**：提供产品比较、在线投保和理赔协助的技术平台。

### 核心资源
- **保险经纪牌照**：合法从事保险经纪业务的资质。
- **产品供给**：与多家保险公司合作获得的产品池。
- **核保/理赔专业能力**：帮助用户理解条款、协助理赔的团队。

### 核心诉求
- **投保人**：找到性价比高、保障合适的保险，投保流程简单，理赔顺利。
- **保险经纪人**：获得客户信任，完成销售并获得佣金。
- **平台**：做大保费规模，通过佣金和增值服务变现。

### 关键矛盾
- **经纪人中立性 vs 佣金导向**：经纪人应站在投保人立场，但佣金由保险公司支付存在利益冲突。
- **条款复杂度 vs 用户理解力**：保险合同条款专业且长，用户难以完全理解，容易产生理赔纠纷。
- **健康告知与承保率**：严格健康告知会导致更多拒保，平台转化率下降。
- **线上服务与线下需求**：标准化产品适合线上，复杂险种需要线下沟通和信任建立。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：在线比价超市模式
- **场景描述**：平台汇集多家保险公司产品，用户可以并排比较保障内容和价格。
- **关键特性**：信息透明度高、用户自主决策、适合标准化程度高的产品。

### 形态二：顾问式经纪人模式
- **场景描述**：用户提交需求后，专业经纪人提供一对一的方案推荐和投保指导。
- **关键特性**：服务质量高、适合复杂产品和高净值客户、人力成本高。

### 形态三：场景嵌入式保险模式
- **场景描述**：保险产品嵌入到其他消费场景中（如买机票同时买航延险）。
- **关键特性**：转化率高、产品碎片化、与场景方的分成是关键。

### 形态四：互助保险模式
- **场景描述**：用户加入互助计划，共同分摊理赔金额，平台收取管理费用。
- **关键特性**：价格低廉、获客速度快、非持牌经营有合规风险。
