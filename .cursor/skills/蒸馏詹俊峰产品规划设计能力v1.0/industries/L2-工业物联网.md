# L2 - 工业物联网

## L1 领域核心模型

### 核心角色
- **工厂运维人员**：监控设备运行状态，处理异常报警，执行维修任务。
- **工厂管理者**：关注设备综合效率（OEE）、能耗和产量。
- **设备制造商**：需要远程监控已售设备状态，提供预测性维护服务。
- **平台运营方**：提供设备连接、数据采集和分析的技术平台。

### 核心资源
- **设备数据**：温度、振动、转速、电流等传感器采集的实时参数。
- **协议适配能力**：兼容 Modbus、OPC UA、MQTT 等多种工业协议。
- **规则引擎**：根据设备参数触发预警和工单的自动化规则。

### 核心诉求
- **运维人员**：及时发现设备异常，减少突发停机，提高维修效率。
- **管理者**：提高设备利用率和产量，降低能耗和维护成本。
- **设备制造商**：从卖设备转型为卖设备+服务，通过预测性维护获得持续收入。
- **平台**：通过设备连接数、存储和分析服务变现。

### 关键矛盾
- **数据采集与改造成本**：老旧设备没有传感器和联网能力，加装改造成本高。
- **数据安全与云化**：工厂担心核心工艺数据上云后被泄露，倾向本地部署。
- **报警风暴与重要遗漏**：规则设置过于敏感导致大量无效报警，运维人员疲于应对。
- **标准协议与碎片化**：不同品牌设备使用不同协议，统一采集难度大。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：设备联网与监控模式
- **场景描述**：通过网关采集设备数据，在云平台显示实时状态和历史趋势。
- **关键特性**：基础能力，解决设备“黑箱”问题。

### 形态二：预测性维护模式
- **场景描述**：基于设备历史数据和故障模型，预测关键部件的剩余寿命。
- **关键特性**：价值更高，直接降低非计划停机，但需要积累大量故障数据。

### 形态三：数字孪生模式
- **场景描述**：在虚拟空间中构建设备或产线的三维数字孪生体，实时映射物理世界。
- **关键特性**：可视化和仿真能力强，适合复杂产线和高端设备。

### 形态四：能耗管理优化模式
- **场景描述**：聚焦能源消耗的采集、监控和分析，识别节能机会。
- **关键特性**：与碳达峰碳中和政策趋势契合，投资回报可量化。
