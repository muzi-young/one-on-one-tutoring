# I5 - 游戏陪玩

## L1 领域核心模型

### 核心角色
- **老板/玩家**：需要陪玩服务，希望有技术指导或社交陪伴的游戏玩家。
- **陪玩师**：提供游戏内带打、教学或陪伴社交的服务者。
- **平台运营方**：匹配老板和陪玩师，提供语音工具和结算担保。

### 核心资源
- **陪玩师供给**：有特定游戏高段位或特殊社交技能的服务者。
- **语音互动**：游戏过程中的实时语音沟通是服务的核心交付方式。
- **信任与安全**：老板和陪玩师之间的人身安全、隐私保护和防骚扰机制。

### 核心诉求
- **老板**：游戏体验更好，上分更快，获得有质量的陪伴和娱乐。
- **陪玩师**：通过技能或聊天获得收入，找到优质稳定的老板。
- **平台**：最大化订单规模和抽佣，管理陪玩师质量，防止涉黄或诈骗。

### 关键矛盾
- **技能陪玩 vs 软色情擦边球**：部分陪玩演变为灰色服务，导致平台受严厉监管。
- **跳单与平台抽佣**：老板和陪玩师私下加微信交易，平台抽佣名存实亡。
- **陪玩师人身安全与老板责任**：线下约会、上门陪玩引发的恶性事件造成社会舆论压力。
- **陪玩质量与匹配效率**：老板要匹配到又强又有趣的陪玩师，高频匹配失败导致流失。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：C2C 陪玩交易平台
- **场景描述**：陪玩师挂靠平台，老板下单，服务结束后评价付款。
- **关键特性**：体量依赖陪玩师数量和老板需求，容易出现头部陪玩垄断流量。

### 形态二：工会/俱乐部派单模式
- **场景描述**：陪玩师加入公会，由公会向平台和老板统一派单、统一抽成。
- **关键特性**：统一管理服务质量，但陪玩师被公会压榨剩余价值。

### 形态三：技术教学专项陪玩
- **场景描述**：只做高分段教学和战术复盘，严格限制社交内容。
- **关键特性**：价格高，合规风险低，适合实力型陪玩师。

### 形态四：语音互动社交陪玩
- **场景描述**：更偏向于玩任何游戏或甚至不玩游戏，以语音聊天和陪伴为主。
- **关键特性**：与语音社交产品界限模糊，社交属性大于游戏属性。
