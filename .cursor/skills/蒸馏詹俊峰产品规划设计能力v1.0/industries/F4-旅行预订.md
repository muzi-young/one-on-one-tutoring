# F4 - 旅行预订

## L1 领域核心模型

### 核心角色
- **旅行者**：预订机票、酒店、火车票、门票等旅游产品的个人或团体。
- **供应商**：航空公司、酒店集团、景区等提供旅游产品的商家。
- **平台/OTA**：聚合产品信息，提供搜索比较、预订支付和售后的中间方。

### 核心资源
- **产品库存**：来自各类供应商的实时库存和价格。
- **用户评价**：大量真实的入住和出行评价是决策的关键参考。
- **供应链关系**：与大型航司和酒店集团的直连或批发协议。

### 核心诉求
- **旅行者**：一站式预订行程，价格透明有优势，遇到变动方便退改。
- **供应商**：提高客房或座位的利用率，降低空置和空座成本。
- **平台**：最大化 GMV 和佣金收入，控制用户投诉和退改纠纷。

### 关键矛盾
- **即时确认与库存同步**：用户要即时确认，但供应商系统的库存同步可能延迟导致超售。
- **低价格与供应商关系**：平台补贴低价会干扰供应商的定价体系，引起供应商不满和断供。
- **行前变动与退改损失**：旅行计划变动频繁，平台和供应商在退改政策上容易与用户冲突。
- **流量集中与分销成本**：供应商依赖平台流量，但佣金率持续上升让供应商利润微薄。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：综合 OTA 模式
- **场景描述**：聚合机票、酒店、火车票、门票等多品类的一站式旅行平台。
- **关键特性**：用户覆盖面广，交叉销售能力强，供应链管理复杂。

### 形态二：垂直搜索/比价模式
- **场景描述**：聚合多家 OTA 和供应商的价格，用户比价后跳转购买。
- **关键特性**：轻资产，重流量获取，变现依赖点击和广告费。

### 形态三：内容+预订模式
- **场景描述**：以游记、攻略、短视频等内容驱动灵感，引导用户预订。
- **关键特性**：决策路径长但转化质量高，适合年轻自由行用户。

### 形态四：商旅管理模式
- **场景描述**：为企业提供差旅预订和费用管理工具。
- **关键特性**：B2B 收入稳定，强调审批流和费用控制，与企业系统深度绑定。
