# C4 - 股票交易

## L1 领域核心模型

### 核心角色
- **投资者/交易者**：买卖股票的个人或机构。
- **券商**：持有证券经纪牌照，为投资者提供交易通道和服务。
- **交易所**：提供股票交易撮合的场所，如沪深交易所、港交所。
- **行情数据商**：提供实时或延时市场数据的供应商。

### 核心资源
- **证券经纪牌照**：合法从事证券经纪业务的资质。
- **交易系统稳定性**：高并发、低延迟的交易执行能力。
- **行情数据**：实时准确的股票行情是交易的基础。

### 核心诉求
- **投资者**：快速执行交易，获取行情和分析工具，交易成本低。
- **券商**：扩大客户规模和交易量，赚取佣金和其他增值服务收入。
- **平台**：提升用户活跃度和留存，通过佣金、两融利息、投顾等变现。

### 关键矛盾
- **交易刺激与风险承受**：便捷的交易工具可能鼓励过度交易，导致亏损。
- **信息获取与内幕交易**：信息优势是交易的核心，但内幕交易违法。
- **券商收入与投资者成本**：佣金是券商传统收入，但佣金战让券商利润变薄。
- **国际化与监管差异**：跨境股票交易面临不同市场的监管规则和时区差异。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：全能型券商模式
- **场景描述**：提供行情、交易、研报、理财等全链条服务。
- **关键特性**：功能全面、用户覆盖广、运营成本高。

### 形态二：互联网折扣券商模式
- **场景描述**：以极低佣金或零佣金吸引用户，通过利息、数据等增值服务变现。
- **关键特性**：获客速度快、产品体验好、对传统券商形成降维打击。

### 形态三：社交跟投模式
- **场景描述**：用户可以查看和复制其他投资者的交易策略。
- **关键特性**：降低新手决策门槛，跟投风险需要充分披露。

### 形态四：量化交易平台模式
- **场景描述**：提供 API 接口和算法交易工具，服务量化投资者。
- **关键特性**：用户专业度高、交易频率高、对系统性能和稳定性要求极高。
