# L4 - 智慧能源

## L1 领域核心模型

### 核心角色
- **用能方**：工厂、商场、居民等电力用户，关注电费和用电稳定性。
- **供能方**：电网公司、售电公司、分布式能源站。
- **设备/系统集成商**：安装光伏、储能、充电桩等能源设备的服务商。
- **平台运营方**：提供能耗监测、能效分析和能源交易的技术平台。

### 核心资源
- **能源数据**：电表、水表、气表等计量器具的实时和累计数据。
- **设备资产**：光伏板、储能电池、充电桩、暖通设备等。
- **电价和政策信息**：分时电价、需求响应补贴、碳交易价格等。

### 核心诉求
- **用能方**：降低电费支出，保障用电安全，满足合规和 ESG 要求。
- **供能方**：提高电网运行效率和设备利用率，降低峰谷差。
- **平台**：通过硬件和 SaaS 服务变现，参与电力交易和需求响应获得分成。

### 关键矛盾
- **设备投入与回报周期**：光伏和储能的初始投入大，回报周期长，中小企业决策犹豫。
- **数据精度与改造成本**：精准到设备级的能耗监测需要改造线路加装传感器，成本高。
- **电力系统安全与开放**：电网公司需要稳定运行，外部平台接入有严格安全要求。
- **政策依赖度高**：补贴退坡或电价政策变动，直接影响项目经济性。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：能耗监测与分析模式
- **场景描述**：安装智能电表，采集能源数据，提供可视化看板和异常报警。
- **关键特性**：最基础的能力，帮用户看清用能情况。

### 形态二：微电网/综合能源管理模式
- **场景描述**：管理分布式光伏、储能和柔性负荷，在分时电价下自动优化用电策略。
- **关键特性**：智能化程度更高，直接创造电费节省的价值。

### 形态三：碳管理/碳排放追踪模式
- **场景描述**：追踪企业碳排放数据，辅助碳交易和 ESG 报告。
- **关键特性**：合规驱动，政策确定性进一步增强后市场爆发。

### 形态四：虚拟电厂/需求响应聚合模式
- **场景描述**：聚合大量分布式资源参与电网的需求响应，获得补偿收入。
- **关键特性**：技术门槛高，需要与电网公司深度合作。
