# H7 - 播客

## L1 领域核心模型

### 核心角色
- **听众**：订阅和收听播客节目的个人，通常在驾车、通勤、做家务时收听。
- **主播/制作人**：策划、录制和发布播客节目的个人或团队。
- **平台运营方**：提供托管、分发、发现和广告投放服务。

### 核心资源
- **原创内容**：独家或优质播客节目是平台差异化的基础。
- **分发网络**：确保节目在全平台都能被订阅和收听到。
- **听众数据**：收听时长、完播率、跳过行为等，用于广告精准投放。

### 核心诉求
- **听众**：发现感兴趣的节目，收听体验流畅，广告不过度打扰。
- **主播**：获得听众和收入（广告分成、听众赞助、付费订阅）。
- **平台**：扩大用户规模和收听时长，通过广告和付费内容变现。

### 关键矛盾
- **创作门槛与内容质量**：播客录制门槛低，但做出高质量节目需要时间和专业能力。
- **商业化与听众体验**：广告是主要收入，但过多广告会让听众流失。
- **开放生态与平台独占**：主播希望多平台分发最大化受众，平台希望独家内容获取竞争优势。
- **长尾内容与头部集中**：听众注意力集中在少数头部节目，新节目难以获得初始流量。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：开放分发平台模式
- **场景描述**：提供 RSS 托管和全平台分发，不独占内容。
- **关键特性**：内容多样，创作者自由度高，平台变现能力较弱。

### 形态二：独家原创内容模式
- **场景描述**：平台投资制作独家播客，吸引用户使用该平台收听。
- **关键特性**：内容深度绑定平台，类似于长视频的 SVOD 模式。

### 形态三：付费订阅/会员模式
- **场景描述**：听众按月付费，可收听独家节目或提前收听、无广告等服务。
- **关键特性**：创作者获得更稳定收入，平台抽成，适合忠实粉丝群体。

### 形态四：嵌入式播客广告网络模式
- **场景描述**：平台不直接面向听众，而是整合广告主和播客，做广告自动插播和收益分配。
- **关键特性**：B2B 属性强，技术驱动的广告匹配和投放效率是核心。
