# F7 - 货运物流

## L1 领域核心模型

### 核心角色
- **发货人/货主**：需要运输货物的企业或个人。
- **司机/车队**：拥有运输车辆，提供运输服务的个体或公司。
- **收货人**：接收货物的最终客户或仓库。
- **网络货运平台**：提供车货匹配、运单管理、支付结算的技术方。

### 核心资源
- **运力池**：注册的车辆和司机数量、类型（车型、载重、长度）和覆盖区域。
- **货源信息**：货主发布的运输需求，是吸引司机的核心。
- **运单数据**：真实的交易流和轨迹，是网络货运平台合规的底线。

### 核心诉求
- **发货人**：快速找到靠谱便宜的车，货物安全准时到达，跟踪有效。
- **司机**：有货可拉，运费高，结款快，不空驶，不拖欠。
- **平台**：撮合更多交易，通过运费差价、抽佣或增值服务变现。

### 关键矛盾
- **信息部/熟人交易壁垒**：司机和货主通过熟人和信息部交易，跳过平台抽成。
- **运费拖欠与平台垫付**：货主习惯赊账，司机需要及时结款，平台是否垫付决定资金压力。
- **责任与追偿**：货损、迟延和事故的责任方难以划分和追偿，司机通常处于弱势。
- **运输质量与低价优先**：平台竞价导致低价中标，服务质量和安全意识无法保证。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：车货匹配信息模式
- **场景描述**：货主发单，车主报价或抢单，线上成交。
- **关键特性**：偏向信息撮合，不介入运输责任和资金流。

### 形态二：网络货运平台模式
- **场景描述**：平台作为承运人签订运输合同，向上游货主开票，向下游司机支付运费。
- **关键特性**：承担运输责任，合规门槛高，需要网络货运牌照。

### 形态三：专线/整车模式
- **场景描述**：只做整车运输，按车型和里程一口价。
- **关键特性**：交易简单，客单价高但频次低，适合大宗货物。

### 形态四：城配/落地配模式
- **场景描述**：城市内短途配送，特别是商超补货和快递落地。
- **关键特性**：高频次，调度难，对时效和司机服务态度要求高。
