# F1 - 网约车

## L1 领域核心模型

### 核心角色
- **乘客**：需要从A点到B点的出行服务的个人。
- **司机**：提供出行服务的车辆驾驶者。
- **平台运营方**：匹配乘客和司机，提供定价、导航、支付和安全保障。

### 核心资源
- **运力供给**：在线可接单的司机数量和分布密度。
- **实时位置数据**：乘客位置、司机位置、车辆行驶状态。
- **定价算法**：动态调价、预估价格和里程计算的算法模型。

### 核心诉求
- **乘客**：叫车快、价格合理、安全到达、司机态度好。
- **司机**：单多、收入高、不被恶意差评、平台抽成合理。
- **平台**：最大化订单规模和匹配效率，控制安全和合规风险，通过抽佣变现。

### 关键矛盾
- **乘客等待与司机空驶**：乘客想快速叫到车，司机想减少空驶等待，供需时空错配是核心难题。
- **高峰运力与闲时运力**：高峰期运力不足，闲时司机闲置，调度弹性是运营难点。
- **安全与效率**：严格的安全审核和行程监控提升安全但增加摩擦。
- **平台抽佣与司机收入**：平台需要抽佣维持运营，司机希望抽佣越低越好。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：快车/专车模式
- **场景描述**：以私家车为主的社会运力，分为经济型快车和品质型专车。
- **关键特性**：运力弹性大，司机门槛适中，是平台的主力业务。

### 形态二：拼车/顺风车模式
- **场景描述**：多位顺路乘客共乘一辆车，分摊费用。
- **关键特性**：乘客省钱，司机提高座位利用率，匹配算法要求更高。

### 形态三：出租车网约化模式
- **场景描述**：传统出租车接入网约平台，线上接单+线下扬招并行。
- **关键特性**：司机有双证合规优势，平台拓展运力池。

### 形态四：企业用车模式
- **场景描述**：企业为员工出行统一采购用车服务，月结付费。
- **关键特性**：B2B 收入稳定，对发票、审批流程和权限管理有额外要求。
