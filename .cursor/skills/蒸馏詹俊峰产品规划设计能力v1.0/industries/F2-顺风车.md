# F2 - 顺风车

## L1 领域核心模型

### 核心角色
- **车主**：有固定出行路线，愿意顺路搭载乘客分摊成本的私家车主。
- **乘客**：寻找性价比高、路线匹配的出行方式的用户。
- **平台运营方**：匹配车主和乘客，提供路线匹配、费用分摊计算和信任保障。

### 核心资源
- **路线数据**：车主的通勤路线和出行时间规律。
- **匹配算法**：基于路线的顺路程度和绕路代价的评估。
- **社区信任**：用户评价、实名认证等信任机制。

### 核心诉求
- **车主**：分摊油费/路费，有人聊天解闷，不影响原有出行计划。
- **乘客**：比网约车便宜，比公交舒适，时间可预期。
- **平台**：降低合规风险，控制安全事件，保持轻资产运营。

### 关键矛盾
- **非营运属性与监管边界**：顺风车需保持非营运性质，运营频次和价格过高会被视为非法营运。
- **匹配率与体验**：严格限制绕路距离保障车主体验，但降低乘客匹配成功率和等待时长。
- **信任与交互摩擦**：陌生人共乘的信任建立、迟到爽约、车内氛围等难以标准化管控。
- **安全与便捷**：安全审核和紧急联系人让乘客放心，但也增加了使用步骤。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：通勤顺风车模式
- **场景描述**：以日常上下班通勤路线为核心，车主和乘客固定匹配。
- **关键特性**：用户粘性高，匹配关系稳定，频次高。

### 形态二：长途拼车模式
- **场景描述**：城市间或跨城长途出行拼车，分摊高速费和油费。
- **关键特性**：客单价高，节假日需求暴增，与高铁大巴形成替代竞争。

### 形态三：社区邻里拼车模式
- **场景描述**：同一小区或周边几公里的短途互助出行。
- **关键特性**：熟人社交感强，信任成本低，适合接送孩子和老人场景。

### 形态四：企业班车共享模式
- **场景描述**：企业间员工共享通勤巴士，平台匹配路线和预定座位。
- **关键特性**：B2B2C 属性，车辆利用率提升，早晚高峰运力固定。
