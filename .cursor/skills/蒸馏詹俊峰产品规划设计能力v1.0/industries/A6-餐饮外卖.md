# A6 - 餐饮外卖

## L1 领域核心模型

### 核心角色
- **消费者**：点餐的用户。
- **商家**：餐厅/外卖档口。
- **骑手**：配送员，将餐品从商家送到消费者手中。
- **平台运营方**：匹配三方，提供订单调度和配送基础设施。

### 核心资源
- **本地商家供给**：足够密度和丰富度的可选餐品。
- **即时配送网络**：能在答应时间内完成配送的骑手运力。
- **位置数据**：消费者位置、商家位置、骑手实时位置。

### 核心诉求
- **消费者**：快速吃到热饭，选择多，配送费合理，遇到问题有赔偿。
- **商家**：获得订单和曝光，控制平台抽佣比例，出餐不被催。
- **骑手**：合理派单和路线，赚到钱，不被差评冤枉。
- **平台**：最大化单量规模，降低每单配送成本，平衡三方利益。

### 关键矛盾
- **快与安全**：消费者要快，但骑手过快可能交通事故，平台需要平衡。
- **佣金与商家利润**：平台抽佣是主要收入，但过高会让商家无利可图。
- **运力弹性**：高峰期运力不足，闲时骑手闲置，调度难度大。
- **体验一致性**：餐品质量取决于商家，配送体验取决于骑手，平台难以完全控制。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：平台派单模式
- **场景描述**：平台系统根据位置和预计时间，自动将订单指派给最合适的骑手。
- **关键特性**：效率最高、骑手被动接单、需要强大的调度算法。

### 形态二：骑手抢单模式
- **场景描述**：订单进入订单池，骑手主动抢单。
- **关键特性**：骑手自由度高、调度不够精准、适合低密度区域。

### 形态三：商家自配送模式
- **场景描述**：平台只负责订单匹配，配送由商家自行解决。
- **关键特性**：平台轻资产、配送体验不可控、适合连锁品牌。

### 形态四：聚合模式
- **场景描述**：平台不自建运力，聚合多家配送服务商供商家选择。
- **关键特性**：技术驱动、对运力的控制力弱于自营配送。
