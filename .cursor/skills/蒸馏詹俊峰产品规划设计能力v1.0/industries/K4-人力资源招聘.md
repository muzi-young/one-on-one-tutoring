# K4 - 人力资源招聘

## L1 领域核心模型

### 核心角色
- **招聘专员/用人经理**：发布职位、筛选简历、安排面试、发放 offer。
- **HR 管理者**：制定招聘计划，管理招聘渠道和预算，分析招聘效率。
- **候选人**：查看职位、投递简历、参与面试。
- **平台运营方**：提供 ATS（招聘流程管理系统）、人才库、多渠道聚合。

### 核心资源
- **职位信息**：职位描述、薪资范围、技能要求。
- **人才数据**：简历库、候选人的面试记录和评价。
- **招聘流程**：从职位申请到入职的标准流程和协作规则。

### 核心诉求
- **招聘方**：高效筛选匹配的候选人，缩短岗位空缺时间。
- **HR 管理者**：降低对猎头和招聘网站的依赖，控制招聘成本，沉淀企业人才库。
- **候选人**：便捷投递简历，及时获得反馈。
- **平台**：通过企业招聘系统付费、简历下载等变现。

### 关键矛盾
- **主动搜索与被动等待**：招聘方希望主动筛选，但主动找候选人费时费力，被动接收简历又可能质量不佳。
- **多渠道发布与集中管理**：招聘方习惯在多个平台发布，但又需要统一管理各渠道简历和进度。
- **数据沉淀与候选人体验**：企业想把所有面试过的人存进人才库持续跟进，但候选人可能反感被长期“养鱼”。
- **内部招聘与公平质疑**：通过系统内推和调动，可能被质疑为“走形式”，实际人选早已内定。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：ATS 招聘管理系统
- **场景描述**：帮企业发布职位、管理简历、安排面试、生成报表。
- **关键特性**：面向HR，提高内部招聘效率。

### 形态二：人才库与主动寻源
- **场景描述**：自带海量人才库，猎头或 HR 可主动搜索和联系候选人。
- **关键特性**：需要持续更新和清洗人才数据，数据库的鲜活性是关键。

### 形态三：AI招聘助手
- **场景描述**：AI自动筛选匹配，生成面试问题，评估面试表现，甚至做背景调查。
- **关键特性**：提升效率，但候选人可能对“机器筛选”有抵触，算法偏见需规避。

### 形态四：一站式招聘外包
- **场景描述**：企业把招聘流程部分或全部外包给平台，按入职结果付费。
- **关键特性**：效果付费，风险更低，但企业对招聘流程的控制力减弱。
