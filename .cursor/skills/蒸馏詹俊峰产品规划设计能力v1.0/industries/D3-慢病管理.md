# D3 - 慢病管理

## L1 领域核心模型

### 核心角色
- **患者**：患糖尿病、高血压、慢阻肺等需长期管理的慢性病用户。
- **医生/健康管理师**：提供长期跟踪、用药指导和生活方式建议的专业人员。
- **平台运营方**：提供数据记录、预警提醒、医患沟通和教育内容的技术方。

### 核心资源
- **患者健康数据**：血糖、血压、体重等日常监测数据。
- **专业管理方案**：针对不同慢病的标准化管理路径和知识库。
- **医患长期关系**：慢病需要长期跟踪，医患信任和依从性是核心。

### 核心诉求
- **患者**：控制病情稳定，减少并发症风险，获得及时的指导和支持。
- **医生/管理师**：提高管理效率，在有限时间服务更多患者。
- **平台**：提升用户长期留存和活跃度，通过服务费、商城、保险等变现。

### 关键矛盾
- **患者依从性与惰性**：慢病管理需要每天坚持，但人性容易懈怠。
- **自动化与个性化**：标准化提醒和管理降低成本，但患者需要个性化的指导。
- **变现难度与刚需属性**：慢病管理刚需，但患者付费意愿需要教育和培养。
- **数据采集与准确度**：智能设备让数据采集更便捷但准确度参差不齐，人工记录准确但麻烦。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：人联驱动管理模式
- **场景描述**：医生或健康管理师通过平台主动随访、调整方案。
- **关键特性**：服务质量高、人力成本大、适合付费意愿强的用户。

### 形态二：工具+内容驱动模式
- **场景描述**：以数据记录工具和科普内容为主，人工干预较少。
- **关键特性**：成本低、覆盖面广、变现依赖广告或商城。

### 形态三：智能硬件+服务模式
- **场景描述**：平台绑定智能血糖仪、血压计等，数据自动上传，异常自动预警。
- **关键特性**：数据连续性好、用户粘性强、硬件成本需要消化。

### 形态四：社群互助模式
- **场景描述**：同病种患者组建社群，互相鼓励分享经验，平台提供管理工具。
- **关键特性**：用户归属感强、社区运营难度大、变现需谨慎。
