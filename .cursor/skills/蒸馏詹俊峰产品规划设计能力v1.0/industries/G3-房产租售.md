# G3 - 房产租售

## L1 领域核心模型

### 核心角色
- **买房/租房者**：寻找住宅、商业地产的个人或企业。
- **卖房/出租方**：房产持有者或开发商。
- **房产经纪人**：撮合交易、带看房屋的中介从业者。
- **平台运营方**：提供房源展示、搜索和经纪人连接工具。

### 核心资源
- **房源数据**：真实的出售和出租房源信息。
- **经纪人网络**：足够数量和密度的经纪人，是线下服务的履约者。
- **数据工具**：估价、房贷计算、市场行情分析。

### 核心诉求
- **买方/租客**：找到真实合适的房子，不被假房源浪费时间。
- **卖方/出租方**：快速出售或出租，获得合理价格。
- **经纪人**：获得客源和房源，提高成交效率。
- **平台**：提高房源和用户的连接效率，通过端口费、广告和交易佣金变现。

### 关键矛盾
- **假房源与平台口碑**：经纪人用低价假房源引流，导致用户对平台不信任。
- **线上引流与线下跳单**：用户通过平台联系经纪人后线下成交，平台无法获得交易佣金。
- **数据产权与隐私**：小区房价和房屋成交价是公共数据，但平台数据可能被爬取。
- **低频交易与服务粘性**：房产交易几年一次，用户买房后基本不再活跃。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：经纪人端口费模式
- **场景描述**：经纪人向平台支付端口费，发布房源并获得用户咨询。
- **关键特性**：平台收入来自 B 端，流量分发规则决定经纪人投入产出。

### 形态二：交易闭环模式
- **场景描述**：从浏览、签约到过户的一站式交易服务。
- **关键特性**：交易佣金高，但合规和线下服务链条长。

### 形态三：直买直租/C2C 模式
- **场景描述**：房东和买家/租客直接联系，平台不通过经纪人介入，只提供信息匹配。
- **关键特性**：减少中介成本，平台背书压力大，需要处理双方欺诈风险。

### 形态四：长租/托管模式
- **场景描述**：平台收房统一装修管理，出租给租客。
- **关键特性**：重资产模式，服务品质可控但资金压力大。
