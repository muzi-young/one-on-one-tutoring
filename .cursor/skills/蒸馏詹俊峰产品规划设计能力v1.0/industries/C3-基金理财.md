# C3 - 基金理财

## L1 领域核心模型

### 核心角色
- **投资者**：购买基金产品、期望资产增值的个人用户。
- **基金公司**：发行和管理基金产品的持牌金融机构。
- **销售平台**：提供基金产品展示、交易和投后服务的渠道方。
- **托管银行**：负责基金资产保管和资金清算的银行。

### 核心资源
- **基金销售牌照**：合法销售基金产品的资质。
- **产品供给**：合作的基金公司和可供选择的基金产品池。
- **投顾能力**：为用户提供资产配置建议的专业能力。

### 核心诉求
- **投资者**：找到适合自己的产品，获得稳定收益，交易成本低，信息透明。
- **基金公司**：扩大管理规模，获得更多资金流入，降低销售费用率。
- **平台**：做大保有量规模，通过销售服务费、投顾费等变现。

### 关键矛盾
- **收益展示与风险揭示**：展示高收益吸引用户购买，但必须充分揭示风险。
- **短期业绩与长期投资**：用户追逐短期业绩导致频繁申赎，损害长期收益。
- **产品丰富度与选择困难**：产品越多用户选择越困难，需要筛选和推荐。
- **投顾中立性与销售利益**：平台有动力推销返佣高的产品，而非最适合用户的产品。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：基金超市模式
- **场景描述**：平台汇集大量基金产品，用户自行搜索、比较和购买。
- **关键特性**：产品丰富、用户自主决策、适合有经验的投资者。

### 形态二：智能投顾模式
- **场景描述**：用户完成风险测评，系统自动推荐基金组合并持续调仓。
- **关键特性**：决策门槛低、人工干预少、适合大众投资者。

### 形态三：定投引导模式
- **场景描述**：平台引导用户设置定期定额投资计划，分散买入成本。
- **关键特性**：培养长期投资习惯，降低市场时机选择难度，提高用户留存。

### 形态四：内容+社区驱动模式
- **场景描述**：通过财经资讯、市场分析和投资者社区，辅助用户做出投资决策。
- **关键特性**：用户粘性高、内容质量是核心竞争力、变现路径更长。
