# E6 - 考试题库

## L1 领域核心模型

### 核心角色
- **考生**：备考特定考试的学生或成人，需要大量练习和错题回顾。
- **题库运营方**：整理真题、模拟题，提供解析和归类。
- **培训机构/老师**：使用题库为学生布置作业和模拟考。

### 核心资源
- **真题+解析**：包含翔实解析的历年真题和模拟题是核心资产。
- **智能批改能力**：对主观题进行AI打分或辅助人工批改。
- **错题本**：自动收集错题并按知识点分类。

### 核心诉求
- **考生**：刷题提分，了解薄弱环节，高效巩固，不错过漏过重要题型。
- **教师**：快速组卷，高效批改，了解班级学情。
- **平台**：通过付费题库、模考和增值服务变现，保持题库更新速度。

### 关键矛盾
- **题海战术与精准练习**：资源有限的学生需要精准练，而大多平台靠题海吸引用户。
- **版权与真题**：历年真题的版权归属存在争议，平台很难解决所有真版权问题。
- **刷题作弊与真实提分**：学生可能只记答案不理解，导致模拟分高但考试分低。
- **人工智能批改主观题**：数学填空和选择题机评容易，作文和简答题精准度和人力成本矛盾。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：通用刷题工具模式
- **场景描述**：大而全的题库，支持多学科、多考试类型，用户自己搜寻和练习。
- **关键特性**：流量变现依赖广告，用户画像泛化。

### 形态二：垂直考试精练模式
- **场景描述**：只做考研、公考、医考等单个垂直赛道，题库质量极高。
- **关键特性**：对单个考试的数据挖掘深，付费率极高。

### 形态三：智能组卷/模考模式
- **场景描述**：AI根据难度系数和知识点分布组卷，结果反馈个人全国排名。
- **关键特性**：交互和反馈感极强，适合考前阶段的冲刺用户。

### 形态四：老师作业工具模式
- **场景描述**：老师通过平台给学生布置线上作业，题库作为备课资源。
- **关键特性**：B端和C端打通，续费决策在老师，学生使用量稳定。
