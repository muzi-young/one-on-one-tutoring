# G2 - 求职招聘

## L1 领域核心模型

### 核心角色
- **求职者**：寻找工作机会的个人，提供自己的简历和技能信息。
- **企业招聘方**：发布职位、筛选简历、安排面试的 HR 或用人经理。
- **平台运营方**：提供职位搜索、匹配推荐和沟通工具的技术方。

### 核心资源
- **简历数据**：求职者的工作经历、学历和技能信息。
- **职位库存**：企业发布的招聘岗位及详细要求。
- **匹配算法**：基于技能标签和经历的推荐引擎。

### 核心诉求
- **求职者**：找到适合自己的工作，投递流程简便，及时得到反馈。
- **企业招聘方**：高效筛选符合条件的候选人，招聘周期短，成本合理。
- **平台**：扩大求职者和职位数量，通过企业付费变现。

### 关键矛盾
- **信息真实性与审核成本**：简历夸大、职位虚假等损害平台信誉。
- **求职者活跃度与招聘冷淡期**：求职者找到工作后消失，使用行为高度低频，留存是长期难题。
- **效率面与多样性**：企业希望筛选快速，但过于依赖硬指标可能漏掉潜在合适的人。
- **企业 B 端付费意愿与 C 端免费**：企业付费是主要收入，但需要足够大的求职者基数支撑。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：综合招聘平台模式
- **场景描述**：覆盖各行业各层级的全品类招聘。
- **关键特性**：用户规模大，匹配效率是核心，收入来自企业套餐和广告。

### 形态二：垂直招聘模式
- **场景描述**：只服务某个行业（如互联网、金融、蓝领），深度满足该行业需求。
- **关键特性**：候选人更精准，企业付费意愿高，天花板在于垂直行业规模。

### 形态三：直聊模式
- **场景描述**：求职者和招聘方可以直接在线沟通，类似聊天式匹配。
- **关键特性**：决策流程短，更适合服务业和蓝领等岗位。

### 形态四：猎头/内推平台模式
- **场景描述**：聚合专业猎头或企业员工，推荐高价值人才获得佣金。
- **关键特性**：高客单价，候选人质量高，人才被动求职者为主。
