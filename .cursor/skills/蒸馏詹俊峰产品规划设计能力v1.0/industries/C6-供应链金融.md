# C6 - 供应链金融

## L1 领域核心模型

### 核心角色
- **融资企业**：供应链中需要融资的中小企业（供应商或经销商）。
- **核心企业**：供应链中信用较强的龙头企业，为上下游融资提供信用背书。
- **资金方**：提供融资资金的银行、保理公司或平台自有资金。
- **平台运营方**：验证贸易真实性、评估信用风险、管理资金流的技术服务方。

### 核心资源
- **核心企业信用**：核心企业的信用是供应链金融的锚，融资成本取决于核心企业资质。
- **贸易数据**：采购订单、发票、入库单、物流单等证明贸易真实性的数据。
- **风控能力**：识别虚假交易和重复融资的能力。

### 核心诉求
- **融资企业**：快速获得低成本资金，改善现金流，扩大经营规模。
- **核心企业**：供应链稳定，供应商和经销商不因资金问题掉链子。
- **资金方**：获得风险可控的资产和收益，降低对单一企业信用的依赖。

### 关键矛盾
- **贸易真实性与造假风险**：虚假贸易和重复融资是供应链金融最大的风险。
- **核心企业配合度**：核心企业需要确认债务，但可能不愿意承担额外责任而消极配合。
- **数据获取与系统对接**：需要和企业ERP、物流系统打通，系统对接成本高。
- **单一核心企业依赖**：风险集中在核心企业，一旦核心企业出问题影响整个链条。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：应收账款融资模式
- **场景描述**：供应商将对核心企业的应收账款转让给资金方，提前获得资金。
- **关键特性**：核心企业确权是关键，风险相对较低，适合有稳定账期的供应链。

### 形态二：预付款融资模式
- **场景描述**：经销商向核心企业采购时，资金方代为支付预付款，经销商分期还款。
- **关键特性**：需要货物控制和处置能力，风险比应收融资高。

### 形态三：存货质押融资模式
- **场景描述**：企业将库存商品质押给资金方获取融资。
- **关键特性**：需要监管货物的能力和快速处置能力，适合大宗商品。

### 形态四：数据信用融资模式
- **场景描述**：基于企业经营数据（开票、纳税、交易流水）直接授信，无需核心企业确权。
- **关键特性**：覆盖面广，对数据质量要求高，风控模型是核心竞争力。
