# A4 - 社区团购

## L1 领域核心模型

### 核心角色
- **团长**：社区内的关键人物，负责建群、推广商品、收货和分发。
- **消费者**：社区居民，通过团长获取低价商品。
- **平台运营方**：选品、采购、仓储、配送，管理团长网络。
- **供应商**：提供货源的品牌或批发商。

### 核心资源
- **社区信任**：团长与社区居民之间的邻里信任关系。
- **集单能力**：将分散的个人订单合并为批量订单，降低履约成本。
- **配送网络**：从中心仓到网格仓到团长自取点的三级物流体系。

### 核心诉求
- **消费者**：买到比超市更便宜的商品，就近取货方便快捷。
- **团长**：赚取佣金，维护好与邻居的关系，不被售后问题困扰。
- **平台**：降低获客成本和配送成本，提升单团密度，控制团长流失率。

### 关键矛盾
- **低价与品质**：消费者对低价的追求可能与商品品质产生冲突。
- **团长的忠诚度**：团长是核心资产，但多平台竞争让团长容易跳槽。
- **效率与控制**：集单配送效率高，但需要强大的供应链和物流控制力。
- **预售与现货**：预售模式降低库存风险，但消费者可能因等待时间长而不满。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：次日自提模式
- **场景描述**：用户当天下单，次日商品到达团长自提点，用户自行取货。
- **关键特性**：履约成本最低、SKU 集中、时效要求高。

### 形态二：即时配送模式
- **场景描述**：用户下单后 30-60 分钟配送到家，团长角色弱化为前置仓。
- **关键特性**：时效快、履约成本高、SKU 更偏生鲜熟食。

### 形态三：批发式团购
- **场景描述**：团购商品以大包装、高性价比为主，面向大家庭囤货需求。
- **关键特性**：客单价高、SKU 少而精、频率较低。

### 形态四：团长自营模式
- **场景描述**：团长自行选品和定价，平台仅提供供应链和物流支持。
- **关键特性**：团长自主权大、平台控制力弱、毛利分配更偏向团长。
