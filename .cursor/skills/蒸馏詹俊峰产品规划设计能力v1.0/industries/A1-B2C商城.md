# A1 - B2C商城

## L1 领域核心模型

### 核心角色
- **消费者**：浏览、搜索、下单、支付、收货、评价的个人用户。
- **平台运营方**：负责商品审核、商家管理、流量分发、售后服务规则制定。
- **商家/品牌方**：入驻平台，发布商品，管理库存，处理订单，提供售后。
- **物流服务方**：承接商品从商家到消费者的配送任务。

### 核心资源
- **商品**：实体或虚拟商品，具有类目、属性、规格（SKU）、价格、库存等特征。
- **流量**：消费者注意力，是平台的核心变现资源。
- **信任**：消费者对商品质量、商家信誉、平台保障的信任。
- **数据**：消费者行为数据、交易数据、评价数据。

### 核心诉求
- **消费者**：找到想要的商品，以合理价格购买，快速安全收到，售后有保障。
- **商家**：获得曝光和订单，管理库存和发货，控制营销成本，积累好评。
- **平台**：最大化交易规模（GMV），提升用户粘性，控制假货/欺诈风险，获取佣金或广告收入。

### 关键矛盾
- **丰富度与决策效率**：商品越多，消费者选择越困难。
- **开放与品控**：降低入驻门槛增加供给，但可能引入低质商家和假货。
- **个性化与隐私**：精准推荐需要用户行为数据，但可能引发隐私担忧。
- **促销依赖**：过度依靠打折促销驱动GMV，会侵蚀利润和品牌价值。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：开放平台模式
- **场景描述**：大量第三方商家入驻，平台提供流量和交易工具，轻资产运营。
- **关键特性**：商家自主运营、SKU丰富、竞争激烈、平台主要靠广告和佣金盈利。

### 形态二：自营模式
- **场景描述**：平台自行采购商品，直接向消费者销售。
- **关键特性**：品控强、毛利高、库存压力和资金占用大、退换货流程统一。

### 形态三：品牌直营（DTC）
- **场景描述**：品牌方在平台上开设官方旗舰店，直接触达消费者。
- **关键特性**：正品保障强、品牌自主定价和营销、与平台分成模式协商空间大。

### 形态四：社交裂变驱动
- **场景描述**：通过拼团、分享红包、分销返佣等方式驱动用户拉新。
- **关键特性**：获客成本低、用户增长快、可能影响品牌调性、需要强风控防止羊毛党。
