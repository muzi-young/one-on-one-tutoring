# G4 - 汽车服务

## L1 领域核心模型

### 核心角色
- **车主**：拥有汽车的个人或企业，需要保养、维修、保险、卖车等服务。
- **服务商家**：4S 店、维修厂、二手车商、保险公司等。
- **平台运营方**：聚合服务商家，提供比较预约和交易撮合。

### 核心资源
- **服务商家网络**：覆盖足够地理范围和品牌车型的服务供应端。
- **车型数据**：车辆品牌、型号、年份、配置参数的结构化数据库。
- **车主数据**：车辆购买时间、行驶里程、保养记录等。

### 核心诉求
- **车主**：找到靠谱、价格合理的服务，不花冤枉钱，不遇到小病大修。
- **服务商家**：获得客源，提高工位利用率和订单。
- **平台**：撮合交易，通过佣金和广告变现。

### 关键矛盾
- **信息不对称与信任**：车主不懂车，服务商容易利用信息不对称过度维修。
- **价格透明与品质保障**：强调比价可能引发商家之间的恶性竞争，导致降低服务品质。
- **低频消费与留存**：保养半年一次，维修是偶发事件，用户打开 App 频次很低。
- **新能源与传统燃油**：新能源车与传统车的服务体系完全不同，平台面临迭代抉择。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：4S 保养维修平台模式
- **场景描述**：聚合 4S 店和维修厂，用户比较价格和服务后预约。
- **关键特性**：增强商家曝光，平台聚合需求后增加了议价能力。

### 形态二：二手车交易模式
- **场景描述**：为个人或商家提供二手车估价、检测和买卖匹配。
- **关键特性**：车况检测环节建立信任，单车交易额高。

### 形态三：车险比价平台模式
- **场景描述**：输入车辆信息，多家保险公司同时报价。
- **关键特性**：降低车主选择成本，有效获取保险佣金和线索费。

### 形态四：加油充电网络聚合模式
- **场景描述**：聚合加油站和充电站，实时显示油价和空闲充电桩。
- **关键特性**：刚需高频，移动支付和预充值购油增加用户黏性。
