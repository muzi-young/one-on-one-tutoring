# E1 - K12学科辅导

## L1 领域核心模型

### 核心角色
- **学生**：需要提升学科成绩或拓展知识面的中小学生。
- **教师/辅导老师**：提供直播授课、作业批改和学习指导的专业人员。
- **家长**：为孩子的学习付费并关注学习效果。
- **平台运营方**：匹配合适的教师和学生，提供教学工具和课程管理。

### 核心资源
- **教师供给**：有教学经验和学科专长的教师，是教学质量的保证。
- **课程体系**：与学校进度匹配或专项提升的课程内容。
- **学习数据**：学生做题正确率、薄弱知识点等，驱动个性化教学。

### 核心诉求
- **学生**：听得懂、学得会、成绩有提升、学习过程不枯燥。
- **家长**：看到孩子的进步，花费值得，减少辅导作业的负担。
- **教师**：高效完成教学任务，获得合理的课酬，减少行政事务。
- **平台**：做大在读学生规模，通过课时费抽成、课程销售等变现。

### 关键矛盾
- **提分目标与兴趣培养**：家长最关注分数，但过度应试会扼杀学习兴趣。
- **师生匹配与排课效率**：好老师时间有限，匹配不合适影响续费率。
- **教学标准化与因材施教**：标准化课程降低成本，但每个学生薄弱点不同。
- **屏幕时间与视力保护**：在线学习增加屏幕使用时间，家长对视力影响担忧。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：双师大班课模式
- **场景描述**：主讲老师大班直播授课，辅导老师小班跟进答疑和作业。
- **关键特性**：名师利用率最大化，运营成本相对可控，适合学科类辅导。

### 形态二：一对一/小班课模式
- **场景描述**：学生与老师一对一或小组形式上课，互动更深入。
- **关键特性**：个性化程度高，效果更有保证，但价格更高，规模化难度大。

### 形态三：AI 自适应学习模式
- **场景描述**：系统根据学生做题情况自动推送学习内容和练习题。
- **关键特性**：不需要真人老师实时参与，边际成本趋近于零，适合练习巩固。

### 形态四：拍照搜题/题库模式
- **场景描述**：学生拍照上传题目，系统返回答案和解析。
- **关键特性**：工具属性强，高频使用，但容易变成抄作业工具。

### 形态五：自习室/陪伴模式
- **场景描述**：通过视频连麦互相监督自习，或虚拟自习场景营造学习氛围。
- **关键特性**：满足自律需求，用户自发参与感强，下沉学生群体受欢迎。
