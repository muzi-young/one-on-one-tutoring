# I2 - 中重度RPG

## L1 领域核心模型

### 核心角色
- **核心玩家**：愿意投入大量时间和金钱在角色扮演、战斗体验和剧情上的玩家。
- **游戏开发商**：投入大量资金和团队进行游戏制作，追求长线运营和用户生命周期价值。
- **发行商/平台**：负责全球发行、服务器维护和社区运营。

### 核心资源
- **剧情与世界观**：引人入胜的故事、丰富的角色设定、沉浸式的世界构建。
- **角色成长系统**：等级、装备、技能、天赋等复杂的养成体系。
- **社交与公会**：副本、公会战、排行榜等强社交玩法。

### 核心诉求
- **玩家**：体验宏大剧情，获得成长满足感和社交归属感，在排行榜和PVP中证明实力。
- **开发商**：维持高日活和付费率，持续推出新内容和活动以延长游戏寿命。
- **平台**：获得高流水分成，通过活动运营和推广获得更多收入。

### 关键矛盾
- **肝与氪的平衡**：核心玩法依赖时间投入还是金钱投入，需要精巧设计以免逼退非氪玩家。
- **内容消耗速度与开发周期**：玩家消耗内容的节奏远快于开发团队更新速度。
- **数值膨胀与老玩家价值**：不断拉高数值吸引付费，但会导致老副本、老道具淘汰。
- **社交绑定与个人体验**：不参加公会和组队就拿不到核心奖励，个人玩家体验受损。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：MMORPG（大型多人在线）
- **场景描述**：大世界无缝连接，千人同屏，强调公会和野外 PVP/PVE。
- **关键特性**：服务器生态社交粘性强，研发成本极高，需要持续资料片支持。

### 形态二：动作/装备驱动RPG
- **场景描述**：注重战斗手感和数值养成，如暗黑类刷装备玩法。
- **关键特性**：赛季重置生命周期，配装和手法深度决定玩家分层。

### 形态三：二次元抽卡 RPG
- **场景描述**：以角色收集和养成为核心，通过抽卡驱动收入，故事内容高频更新。
- **关键特性**：角色塑造和人设是核心资产，PVE 为主，社区同人内容繁荣。

### 形态四：开放世界/沙盒 RPG
- **场景描述**：没有既定主线，玩家自由探索、建造和交互。
- **关键特性**：对物理引擎和 AI 要求高，UGC 内容往往是游戏生命线的延伸。
