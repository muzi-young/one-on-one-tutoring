# B8 - 游戏社交

## L1 领域核心模型

### 核心角色
- **玩家**：希望通过游戏社交找到玩伴、队友或竞争对手。
- **陪玩师**：提供游戏陪玩服务并收取费用的玩家。
- **公会/战队管理者**：组织和管理游戏内社群的玩家。
- **平台运营方**：提供组队匹配、语音工具和社区管理的平台。

### 核心资源
- **游戏数据**：玩家的游戏技能、战绩、段位等数据，是匹配和社交的核心依据。
- **组队效率**：快速找到合适的队友是游戏社交最重要的功能。
- **社区文化**：围绕特定游戏形成的梗、黑话和互动方式。

### 核心诉求
- **玩家**：快速找到靠谱队友，提升游戏体验，结交游戏朋友。
- **陪玩师**：通过技能和时间换取收入，获得稳定的客源。
- **平台**：提升用户粘性和活跃度，通过陪玩交易抽佣、会员等变现。

### 关键矛盾
- **技能匹配与社交匹配**：技术好的队友脾气不一定好，合得来的人技术不一定行。
- **付费陪玩与公平性**：陪玩提升了付费者的游戏体验，但可能影响其他人的公平。
- **游戏厂商依赖**：游戏社交平台通常依赖第三方游戏，游戏生命周期直接影响平台。
- **社交深度与游戏绑定**：一旦用户换了游戏，基于旧游戏建立的社交关系可能瓦解。

## L2 动态形态参考

*请注意：以下为常见业务形态，AI 可根据需求自由组合、修改或颠覆。*

### 形态一：游戏社区+组队工具
- **场景描述**：平台提供游戏社区讨论、组队招募、战绩展示等功能。
- **关键特性**：用户自发社交，平台提供工具聚合，适合大型游戏生态。

### 形态二：陪玩交易平台
- **场景描述**：陪玩师挂靠平台，用户付费下单，平台担保和抽佣。
- **关键特性**：强交易属性，陪玩师质量是关键，需要评价和纠纷处理机制。

### 形态三：语音开黑+社交模式
- **场景描述**：以语音工具为核心，用户在房间内聊天、组队、玩游戏。
- **关键特性**：语音工具属性强，游戏过程中的沟通体验决定产品口碑。

### 形态四：游戏内社交模块
- **场景描述**：社交能力直接嵌入游戏，如游戏内的好友系统、公会系统。
- **关键特性**：社交是游戏的一部分，由游戏厂商主导，第三方平台无法介入。
