# 通用业务模式库

本文件定义了跨行业的通用业务模式模型。当项目无法匹配特定行业子场景时，AI 从此库提取可复用的业务模式元素，结合用户需求进行定制化组合。

---

## L1 领域核心模型

### 核心交易结构

任何业务都可以拆解为以下基本元素的组合：

- **供给方**：提供产品或服务的一方。
- **需求方**：消费产品或服务的一方。
- **交易对象**：被交换的产品、服务或信息。
- **交易媒介**：促成供需匹配的机制（平台、中介、直销渠道等）。
- **价值交换形式**：货币、时间、注意力、数据、权益等。

### 核心诉求（跨行业通用）

- **效率**：更快地完成交易或任务。
- **成本**：降低获取产品或服务的成本。
- **信任**：在交易双方之间建立可靠的关系。
- **便利**：减少使用摩擦，降低使用门槛。
- **安全**：保护资产、数据、隐私不受侵害。

### 关键矛盾（跨行业通用）

- **规模与质量**：快速扩张与保持服务质量之间的冲突。
- **开放与安全**：让更多用户参与与防止恶意行为之间的平衡。
- **标准化与个性化**：统一流程降低成本与满足个体差异之间的取舍。
- **信息透明与隐私保护**：供需双方需要信息对称，但又需要保护商业敏感数据。

## L2 动态形态参考

*请注意：以下为常见商业模式形态，AI 可根据需求自由组合、修改或颠覆。*

### 模式一：双边平台

- **场景描述**：连接两类或多类不同用户群体，通过平台促成互动或交易。
- **关键特性**：网络效应、供给端和需求端相互促进、冷启动困难、鸡与蛋问题。
- **适用场景**：电商、出行、社交、招聘、房产等。

### 模式二：单边服务/SaaS

- **场景描述**：直接向用户提供软件服务，用户为使用价值付费。
- **关键特性**：订阅制或按量计费、用户留存率决定长期价值、功能深度决定竞争力。
- **适用场景**：企业办公、项目管理、设计工具、AI 写作等。

### 模式三：内容驱动

- **场景描述**：以内容（文章、视频、音频等）吸引用户，通过广告、付费或电商变现。
- **关键特性**：内容质量和更新频率决定用户粘性、流量分配机制影响生态健康。
- **适用场景**：短视频、长视频、播客、数字阅读、知识付费等。

### 模式四：交易撮合

- **场景描述**：为买卖双方提供信息匹配、信任背书和交易支持。
- **关键特性**：信息不对称是核心痛点、佣金或差价是主要盈利方式。
- **适用场景**：大宗贸易、房产经纪、保险经纪、二手车交易等。

### 模式五：社区驱动

- **场景描述**：以用户关系或共同兴趣为核心，靠社区氛围和互动驱动增长。
- **关键特性**：用户产生内容（UGC）、文化和规则自我演化、商业化需要平衡用户体验。
- **适用场景**：兴趣小组、知识问答、游戏公会、粉丝社群等。

### 模式六：数据/API 服务

- **场景描述**：提供数据查询、分析或 API 调用服务，客户按需付费。
- **关键特性**：数据的准确性和实时性决定价值、技术门槛较高、客户替换成本大。
- **适用场景**：风控评分、物流追踪、身份验证、市场行情等。
