# 通用盈利与商业模式

本文件定义了跨行业的常见变现方式和定价策略。当项目无法匹配特定行业子场景时，AI 从此库提取可复用的盈利模式，结合用户需求推荐合适的组合。

---

## L1 领域核心模型

### 盈利模式设计的核心问题

- **谁付费**：终端用户、商户、第三方广告主、还是多方？
- **为什么付**：获得使用权、获得曝光、获得交易机会、还是获得增值服务？
- **怎么定价**：按时间、按用量、按交易额、按效果、还是一口价？
- **什么时候收**：前置收费、后置收费、还是混合模式？

### 定价策略原则

- **价值定价**：价格应反映用户感知到的价值，而非成本。
- **阶梯定价**：不同层级的用户有不同需求和付费意愿，分层定价可最大化收入。
- **免费增值（Freemium）**：用免费版吸引用户，通过高级功能或用量限制推动付费转化。

## L2 动态形态参考

*请注意：以下为常见盈利模式，AI 可根据需求自由组合。一个项目可以同时采用多种模式。*

### 模式一：交易佣金

- **核心逻辑**：从每笔成功交易中抽取一定比例作为平台收入。
- **适用场景**：电商、出行、房产经纪、大宗贸易、众筹等有明确交易闭环的项目。
- **关键设计**：佣金比例、由谁承担（买方/卖方/双方）、是否有封顶或最低收费。

### 模式二：订阅制

- **核心逻辑**：用户按周期（月/年）支付费用，获得持续的服务使用权。
- **适用场景**：SaaS、内容平台、会员服务、在线教育等。
- **关键设计**：订阅档次（基础/高级/企业）、免费试用期、自动续费规则、退款政策。

### 模式三：广告变现

- **核心逻辑**：通过聚集用户注意力，向广告主出售曝光或点击。
- **适用场景**：内容平台、社交平台、搜索工具等高流量产品。
- **关键设计**：广告形式（Banner、信息流、搜索广告）、计费方式（CPM/CPC/CPA）、用户体验平衡。

### 模式四：增值服务

- **核心逻辑**：基础服务免费，高级功能或服务需要付费。
- **适用场景**：工具类产品、游戏、社交平台等。
- **关键设计**：哪些功能免费、哪些付费？免费版的功能边界在哪里？单个付费功能还是一口价打包？

### 模式五：数据分析与报告

- **核心逻辑**：将平台积累的数据加工为有价值的报告或 API，向客户收费。
- **适用场景**：金融数据、市场研究、行业分析等。
- **关键设计**：数据更新频率、报告形式（PDF/在线看板/API）、定价模式（订阅/单次购买）。

### 模式六：认证与背书

- **核心逻辑**：为用户或商户提供认证服务，认证标志增加其可信度，平台收取认证费。
- **适用场景**：招聘、房产、二手交易、B2B 平台等信任成本高的场景。
- **关键设计**：认证流程的严谨度（决定认证价值）、认证有效期、认证费用分层。

### 模式七：金融增值

- **核心逻辑**：基于平台交易数据，提供信贷、垫资、保险等金融服务，赚取利差或手续费。
- **适用场景**：供应链、大宗贸易、电商等有大额资金流动的场景。
- **关键设计**：风控模型、资金来源、合规要求（是否需要牌照）。

### 模式八：白标/技术输出

- **核心逻辑**：将平台的技术能力封装为 SDK 或 API，授权给第三方使用。
- **适用场景**：AI 能力、地图服务、身份验证等。
- **关键设计**：定价（按调用量/包年包月）、技术支持范围、SLA 承诺。
