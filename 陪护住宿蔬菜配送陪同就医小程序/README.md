## 四、客户演示脚本（4 分钟）

### 演示前准备

1. 双击 `index.html` 打开总览页（保持开启）
2. 在浏览器分别打开三个原型 HTML（建议 Tab 排序：用户端 → 陪诊师端 → PC 后台）
3. 全屏浏览器、关闭无关窗口

### 演示流程

| 时长 | 端 | 操作要点 | 价值传递 |
|---|---|---|---|
| 30s | 用户端 | 进入首页 → 介绍人物背景：「张先生 67 岁，需异地就诊」 | 一站式入口、适老化 |
| 60s | 用户端 | 住宿预订全链路：列表 → 详情 → 日期 → 确认 → 支付 → 订单 | 房态自动锁定、一键导航 |
| 90s | 用户端 → 陪诊师端 | 用户端预约陪诊并支付后切到陪诊师端「派单任务 / 待服务 → 订单详情 → 上报进度」，再切回用户端「服务进度页」（订单均由后台派单，端内无接单） | 三端实时联动 |
| 60s | PC 后台 | 数据概览 → 待审核陪诊师审核（弹窗） → 待派单订单派单（智能推荐）→ 营收统计排行榜 | 运营闭环、数据可视化 |
| 30s | 用户端 | 个人中心 + 在线客服 → 总结"3 端联动 · 4min 闭环 · 适老化首位" | 收尾 |

### 备用演示路径

如客户希望深入某个场景：
- **强医疗背景客户**：着重陪诊场景 + 陪诊师审核流程
- **强商业地产客户**：着重住宿场景 + 房源/营收管理
- **二期蔬菜配送客户**：可直接基于现有架构添加（详见第八节）

## 五、可编辑点（开发即设计）

所有原型为**单文件 HTML**，所有数据、文案、主题色集中在文件顶部，便于快速修改。

### 常见修改入口

| 想改什么 | 文件位置 |
|---|---|
| **主题色 / 字号 / 圆角** | 顶部 `:root { --xxx: #xxx; }` 变量 |
| **页面文案** | 各 `pages.xxx = ()=>` 函数内的 HTML 字符串 |
| **房源 / 陪诊师数据** | `01-用户端小程序原型.html` 中的 `HOTELS` / `NURSES` / `SERVICES` 数组 |
| **支付反馈 / Toast** | `pay()` / `toast()` 函数 |
| **新增页面** | 在 `pages` 对象中新增 `pages['new-page'] = ()=>({meta, body})` |
| **图表数据** | `03-平台管理后台原型.html` 中各 `chart-bars` / `donut` / SVG 内的内联数据 |

### 添加新页面（示例）

在 `01-用户端小程序原型.html` 中新增"邀请有礼"页面：

```javascript
pages['invite'] = ()=>({
  meta: { title: '邀请有礼', subtitle: '邀请家人享受 8 折优惠' },
  body: `
    <div class="card">
      <h3>邀请记录</h3>
      ...
    </div>
  `
});
```

随后在右侧"全部页面索引"区域增加 `<button onclick="go('invite')">邀请有礼</button>` 即可在导航中出现。
