# 乌兹别克斯坦教育数字化平台 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
乌兹别克斯坦头部教育连锁机构现运营10余家直营门店，业务覆盖语言培训、K12学科辅导与职业技能教育，以3-8人小班课与一对一为主要模式。未来3-5年计划扩展至30-50家门店，但现有数字化基础设施已明显制约连锁化进程。

### 1.2 项目目标
构建一套OMO（Online-Merge-Offline）融合教学数字化平台，采用独立服务器部署与完整源码交付模式，确保客户对核心数据与系统演进的完全自主可控。

### 1.3 核心价值
- 打破时空限制，提升师资利用率
- 沉淀教学数据，实现个性化学习
- 统一多门店管理，提升运营效率
- 支持多语言环境，适配本地化需求

## 2. 用户角色分析

### 2.1 核心用户角色
1. **学员** - 学习服务的最终消费者
2. **教师** - 教学服务的提供者
3. **分店管理人员** - 门店运营管理者
4. **总部管理人员** - 集团战略决策者
5. **家长** - K12学员的决策者和付费者

### 2.2 用户特征
- 多语言环境：中文、英文、俄语、乌兹别克语
- 多时区：UTC+5（塔什干时间）
- 移动优先：89%用户通过手机上网
- 支付习惯：本地电子钱包 + 国际信用卡

## 3. 功能需求

### 3.1 学生端功能

#### 3.1.1 课程中心
**功能描述**：提供课程发现、浏览、体验的核心入口
- **课程分类浏览**：按语言培训、K12学科辅导、职业技能三大类目分类
- **课程详情展示**：大纲、教师简介、评价、价格、已报名人数
- **试听体验**：5-15分钟精选片段，免登录体验
- **搜索筛选**：关键词搜索 + 多维度筛选（价格/时间/语言/类型）

#### 3.1.2 在线学习
**功能描述**：核心教学交付模块
- **录播视频播放**：断点续播、五档倍速、时间轴笔记、画质自适应
- **直播上课**：小班互动（举手/答题/白板）+ 大班讲授（聊天/投票）
- **弱网适配**：动态码率调整，最低480p/1Mbps带宽

#### 3.1.3 学习管理
**功能描述**：学习进度跟踪与管理
- **已购课程**：卡片式列表，按状态分组（进行中/未开始/即将到期）
- **进度追踪**：章节观看时长 + 作业完成率 + 出勤率综合计算
- **学习报告**：周/月度数据摘要，含知识点掌握度雷达图
- **续费提醒**：到期前30/7/1天多通道触达

#### 3.1.4 预约排课
**功能描述**：灵活的课程预约系统
- **时段查看**：14天网格视图，标注可预约/已占用/不可用
- **一键预约**：选择时段 + 上课方式 + 预习需求，10分钟锁定
- **取消/改期**：24小时前免费操作，紧急取消按规则扣费
- **个人课表**：周视图/日视图，支持导出至手机日历

#### 3.1.5 作业考试
**功能描述**：完整的作业与考试管理
- **在线提交**：文字/拍照/附件三种方式，截止前可多次修改
- **成绩查询**：历次得分趋势折线图与详细批改记录
- **错题本**：自动收集错题，按知识点分类，支持相似题重练

#### 3.1.6 个人中心
**功能描述**：用户信息管理与配置
- **信息管理**：基本资料维护、手机号绑定、家长联系方式
- **多语言切换**：俄语/乌兹别克语/英语三语实时切换
- **消息通知**：系统/营销/互动三类通知聚合，支持筛选
- **订单历史**：线上线下订单统一查询，显示门店与顾问信息

### 3.2 教师端功能

#### 3.2.1 课表管理
**功能描述**：教师授课安排管理
- **时间轴视图**：7天彩色卡片视图，按课程类型颜色编码
- **上课提醒**：课前30分钟/5分钟两次推送，附带快捷入口
- **调课申请**：提交变更请求，学生逐一确认后生效

#### 3.2.2 直播授课
**功能描述**：核心教学工具
- **一键开播**：开播前检查清单（设备/网络/课件），一键进入
- **课件上传**：PPT/PDF/Word上传，课前自动转码适配
- **屏幕共享**：全屏或窗口级共享，适用于操作类教学
- **白板书写**：画笔/文本/图形工具，实时同步，支持保存
- **学生互动面板**：举手队列/聊天消息/答题统计，侧边栏常驻
- **课程录制**：开播自动录制，课后2小时转码推送学生

#### 3.2.3 学员管理
**功能描述**：学员学习状态跟踪
- **学员列表**：按课程分组，显示上课次数与最近上课时间
- **进度跟踪**：视频观看/作业/出勤/知识点掌握度四维追踪
- **课后评价**：四维度1-5分评分 + 文字评语
- **班级学情分析**：班级平均进度/正确率热力图/成绩分布

#### 3.2.4 作业批改
**功能描述**：高效的作业处理流程
- **作业布置**：选择班级 + 填写描述 + 上传题目 + 设置截止时间
- **在线批改**：文字批注/图片圈画/客观题自动批改
- **批改状态追踪**：待批改/批改中/已完成状态可视化
- **优秀作业展示**：标记优秀，脱敏后展示给全班

#### 3.2.5 个人中心
**功能描述**：教师职业数据管理
- **课时统计**：按月/按类型统计已完成/待上/调课/请假
- **薪资查看**：课时费 + 绩效奖金 + 补贴扣款逐项明细
- **教学数据统计**：累计时长/学员数/评分/续费率可视化

### 3.3 总部管理后台功能

#### 3.3.1 门店管理
**功能描述**：全部门店运营管理
- **分店信息档案**：门店名称、地址、联系方式、教室数量
- **分店账号创建**：为新店生成独立管理后台账号体系
- **门店运营监控**：关键指标仪表盘，红黄绿三色状态灯

#### 3.3.2 课程管理
**功能描述**：课程内容中台管理
- **课程创建**：结构化表单，支持多语言课程信息录入
- **分类标签体系**：一级类目 + 二级标签 + 自定义标签三级结构
- **定价策略**：固定价格/分期付款/课时包/订阅制多种模型
- **多语言配置**：原文-译文对照界面，支持批量导入导出

#### 3.3.3 教师管理
**功能描述**：师资队伍管理
- **教师档案**：基础信息、教育背景、教学专长、语言能力、资质证书
- **资质审核**：提交→初审→复审→归档四级流程
- **课时统计**：多维度汇总授课数据
- **跨店调度**：教师可授课门店配置与调度管理

#### 3.3.4 学员管理
**功能描述**：学员全生命周期管理
- **全量学员档案**：基础信息、账户信息、关联家长信息
- **学习轨迹**：时间轴展示完整学习行为
- **标签画像**：自动标签 + 手动标签结合的用户分群
- **线索转化跟踪**：从获客到购课的全链路管理

#### 3.3.5 财务管理
**功能描述**：资金流管理与核算
- **总部汇总报表**：日/周/月/季度自动生成营收报表
- **分店营收统计**：各分店独立收支明细
- **应收应付管理**：未结清款项跟踪与催缴
- **分账结算**：总部与分店之间资金分配自动化

#### 3.3.6 营销工具
**功能描述**：标准化营销活动配置
- **优惠券管理**：创建、分发、核销一体化管理
- **拼团活动**：社交裂变营销工具
- **转介绍系统**：推荐奖励机制
- **渠道分析**：各渠道获客效率评估

### 3.4 分店管理后台功能

#### 3.4.1 本地运营管理
**功能描述**：分店日常运营工具
- **本店学员管理**：仅限本店学员数据的查看与管理
- **教师排班**：本店教师的课程安排与调度
- **教室资源调度**：教室使用情况与预约管理
- **营收统计**：本店营收数据与业绩分析

#### 3.4.2 客户关系管理
**功能描述**：分店客户维护
- **线索分配**：新线索分配给销售顾问
- **跟进记录**：客户沟通历史记录
- **转化漏斗**：从线索到付费的转化分析
- **客户服务**：投诉处理与问题解决

## 4. 非功能性需求

### 4.1 性能要求
- **并发能力**：单门店500人同时在线，全局10,000人并发
- **直播延迟**：端到端延迟≤800ms
- **系统可用性**：99.9%年度可用性
- **响应时间**：页面加载时间≤3秒

### 4.2 安全要求
- **数据加密**：HTTPS全链路TLS 1.3，敏感数据AES-256加密
- **访问控制**：基于角色的权限管理（RBAC）
- **数据备份**：日全量 + 6小时增量，异地存储
- **审计日志**：敏感操作记录，保留180天

### 4.3 兼容性要求
- **移动端**：Android 5.0+，iOS 10+
- **浏览器**：Chrome 80+，Safari 13+，Firefox 75+
- **多语言**：中文、英文、俄语、乌兹别克语
- **时区**：UTC+5（塔什干时间）

### 4.4 扩展性要求
- **门店扩展**：支持50+门店扩展
- **用户扩展**：支持100,000+注册用户
- **存储扩展**：支持PB级视频存储
- **功能扩展**：模块化架构支持快速功能迭代

## 5. 技术架构

### 5.1 架构原则
- **微服务架构**：Spring Cloud Alibaba微服务体系
- **容器化部署**：Docker + Kubernetes
- **独立部署**：On-Premises部署模式
- **源码交付**：完整源代码与部署脚本

### 5.2 技术栈
- **后端**：Spring Boot 3.2.x + Spring Cloud Alibaba
- **前端**：Flutter（移动端） + Vue 3 + TypeScript（管理后台）
- **数据库**：MySQL 8.0 集群 + Redis 7.x
- **消息队列**：RabbitMQ 3.12
- **实时通信**：WebRTC + WebSocket

### 5.3 部署要求
- **服务器配置**：生产环境8-10台，16核64GB内存
- **网络带宽**：≥500Mbps双线路由
- **存储要求**：对象存储服务，支持PB级扩展
- **CDN加速**：本地CDN节点，首帧加载≤2秒

## 8. 风险与应对

### 8.1 技术风险
- **风险**：弱网环境下直播质量
- **应对**：多码率自适应 + 前向纠错机制

### 8.2 业务风险
- **风险**：用户接受度不高
- **应对**：分阶段推广 + 培训支持

### 8.3 合规风险
- **风险**：数据本地化合规要求
- **应对**：独立部署 + 数据存储本地化
