## 需求摘要

一套 **半导体设备上位机**，统一接入 **RF 射频电源、匹配器、VI 传感器、DDL 可调负载、Impedance 传感器** 等，支持 **串口 / 以太网 / EtherCAT / USB（DLL）** 多通道、**旧协议兼容** 与 **新协议（握手自描述）** 扩展，并具备 **设备自动扫描识别、Config 驱动界面生成、实时监控与波形绘制、任务编排（毫秒级相对时序）、数据记录（异步存储）、日志诊断、本地数据库查询、权限与审计**（SRC-A + 会议纪要）

## 2. 项目定位与目标

### 2.1 一句话定位（SRC-A）

**半导体设备上位机监控与控制软件**：统一对接 **多类设备**，支持 **多协议通信**、**实时监控**、**任务调度**、**数据记录**、**日志诊断**、**权限管理**。

### 2.2 业务目标（闭环）


| 编号   | 目标                                                     | 验收指向                            |
| ---- | ------------------------------------------------------ | ------------------------------- |
| G-01 | 同一套上位机覆盖 **RF 射频电源、匹配器、VI 传感器、DDL 可调负载、Impedance** 等对象 | 五类设备均有 **连接—查询—设置—记录—曲线** 最小闭环  |
| G-02 | **旧协议兼容 + 新协议可扩展**（握手自描述）                              | 至少各选 **1 台旧机 + 1 台新机** 走通握手与参数树 |
| G-03 | **多设备、多接口** 并行运行，界面不卡、数据可追溯                            | 压测报告 + 日志/库表样本                  |
| G-04 | 支持 **工艺任务编排**（毫秒级相对时序，非日历定时）                           | 任务脚本可保存、可重复执行、可中止               |
| G-05 | **设备即插即用**：连接后自动扫描识别、Config 驱动 UI 生成                   | 扫描进度条反馈 + 识别后设备树自动添加 + 默认界面自动渲染 |
| G-06 | **三期分阶段交付**：一期基础版（绘图+通信）→ 二期进阶版（数学通道+触发）→ 三期完整版        | 每期独立验收、阶段里程碑书面签字                |
| G-07 | **本地离线数据管理**：SQLite 存储历史数据、支持可视化查询、文件拷贝导出              | 百万行查询响应时间达标；导出文件可被另一台机器还原查看     |

## 3. 设备对象与协议矩阵

### 3.1 设备谱系


| 设备类型          | 子类/说明 | 接口形态（SRC-A）            | 协议线索（SRC-B）                                            |
| ------------- | ----- | ---------------------- | ------------------------------------------------------ |
| RF 射频电源       | A 型   | 串口、网口                  | A 型：频率/功率/输出类型/开关…                                     |
| RF 射频电源       | M 型   | 串口、网口（设备为 **TCP 服务端**） | M 型：如 `FRQ`/`OEM`/`PSW0/1`、`DRV1`、`TRG ON/OFF` 等（示例）   |
| 匹配器 Match     | CIP2  | 串口（**Modbus**）         | Modbus：FC10 设置、FC04 查询，**固定寄存器集**（数十个）                 |
| 匹配器           | 通用    | 串口、网口（**快速数据**）        | 通用：模式/电机位置/预设点/配置参数…；查询含 **通用查询报文**、配置查询               |
| 匹配器           | A 型   | 匹配器 A 型协议栈             | 设置：控制模式、电机位置、预设点、电机初始化等 **十六进制示例帧**；查询：基本信息、过程状态等      |
| VI Sensor     | 通用    | 串口、网口                  | **类似 RF M 型**；**HTTP/JSON + VS34** —— **以甲方 ICD 合并定义** |
| DDL 可调负载      | 通用    | 串口（自定义协议2）             | 设置电机位置、查询指令等 **帧结构示例**                                 |
| Impedance 传感器 | 通用    | 网口（优先）、USB（DLL 接口）     | 归属 VI Sensor 大类；原厂协议；USB 方案需调用甲方提供的 DLL                |


### 3.2 新旧设备策略

- **旧设备**：一机一协议，上位机需 **协议兼容层**（专用驱动或配置化适配）。  
- **新设备**：增加与上位机对接的 **统一新协议**（握手返回数据结构，支撑通用 UI）。  
- **EtherCAT**：SRC-A 注明设备多支持 EtherCAT；上位机侧规划 **SOEM 主站**、从站扫描、PDO 字典解析、OP、**1ms 过程数据轮询** 等。

### 3.3 新通用协议

**握手**：上位机发握手报文 → 设备回复：**设备名称、类型、SN、可查询上报数据结构、可设置下发数据结构**（SRC-A）。协议脑图示例口令 `WELCOME TO JOIN US`（SRC-B）—— **实现以冻结 ICD 为准**。

**查询 / 设置**：按握手得到的数据描述，支持 **全量或单项** 查询与设置；SRC-B 提及 **统一全读/全写指令**（字段级待 ICD）。

## 4. 通信模块需求（结构化）

### 4.1 三种通信接口


| 接口            | 能力要点                                                                | 旧设备   | 新设备                  |
| ------------- | ------------------------------------------------------------------- | ----- | -------------------- |
| 串口 SerialPort | **多路**、一对一、**界面可配置通道数**                                             | 既有协议  | 新协议 + **握手**         |
| 以太网 EtherNet  | **一对多**；服务端/客户端模式 **待定**                                            | 既有协议  | 新协议 + **握手**         |
| EtherCAT      | **SOEM 主站**：扫描从站、读 PDO 字典、使能 OP；**PDO** Output/Input；**1ms** 轮询过程数据 | 按从站字典 | 与新协议数据模型 **映射关系待定义** |


### 4.2 与会议录音的交叉说明

- **多串口 + 多网口**、每设备 **查询周期可配**、**多线程**、默认约 **100ms** 轮询、界面 **限帧绘制**（如 ~30fps）且 **数据不丢**（缓冲后绘制）。  
- 与 SRC-A「**快速数据 1ms + UDP**」并列：架构区分 **一般数据路径** 与 **快速数据路径**

## 4A. 设备自动识别与扫描机制（会议纪要 2026-04-22 新增）

> 本章节来源于 2026-04-22 三次客户会议达成的共识，是通信模块的前置核心流程。

### 4A.1 自动扫描与握手识别


| 子编号     | 需求描述                                                                              | 验收要点                       |
| ------- | --------------------------------------------------------------------------------- | -------------------------- |
| SCAN-01 | **Config 文件驱动扫描**：上位机启动后加载所有设备 Config 文件，每份 Config 描述一种设备类型的识别规则（握手指令、期望回复、连接参数等） | Config 文件支持版本号管理；加载失败有明确提示 |
| SCAN-02 | **全端口轮询**：点击「扫描设备」后，上位机依次对所有可用 COM 口和网口发送每份 Config 中定义的握手指令                       | 扫描覆盖所有端口 × 所有 Config 组合    |
| SCAN-03 | **设备型号识别**：根据设备回复的特定字符串（如 Hex 1A）或报文特征码匹配 Config 中的判定规则，确定设备型号                    | 同一端口对多个 Config 逐一尝试；匹配即绑定  |
| SCAN-04 | **自动添加至设备树**：识别成功后，上位机自动将设备添加至左侧设备树，显示在线状态及设备型号；无回复则标记为离线/不存在                     | 识别结果与物理连接一致                |
| SCAN-05 | **扫描进度反馈**：扫描过程中显示进度条，进度按 `端口数 × Config数` 计算                                      | 可估算总耗时；200ms/端口级别扫描速度      |
| SCAN-06 | **多接口优先级**：同一设备同时支持串口和网口时，默认优先尝试串口连接，失败后自动切换网口；优先级顺序客户可配                          | 优先级可在 Config 中定义           |
| SCAN-07 | **设备数量上限**：一般不超过 10 台设备，典型场景 2~3 台                                                | 扫描 10 台设备总耗时 ≤ 约定秒数        |


### 4A.2 Config 文件结构（客户提供，乙方解析）

Config 文件为设备的「身份证 + 通信说明书」，客户方整理提供，至少包含以下字段：


| 字段                  | 说明                    | 示例                                                                     |
| ------------------- | --------------------- | ---------------------------------------------------------------------- |
| `deviceType`        | 设备类型标识                | `"RF-A"`, `"Matcher-CIP2"`, `"VI-Sensor"`                              |
| `deviceName`        | 设备显示名称                | `"RF 射频电源 A 型"`                                                        |
| `interfacePriority` | 接口优先级                 | `["serial", "tcp"]`                                                    |
| `handshake`         | 握手配置：发送指令、期望回复特征、超时时间 | `{ "send": "VERSION", "expectHex": "1A", "timeoutMs": 500 }`           |
| `serialConfig`      | 串口默认参数                | `{ "baudRate": 9600, "dataBits": 8, "stopBits": 1, "parity": "none" }` |
| `tcpConfig`         | 网口默认参数                | `{ "port": 5000, "role": "client", "timeoutMs": 3000 }`                |
| `queryCommands`     | 心跳/查询指令列表（按顺序发送）      | `["HPV", "STATUS", "MODE"]`                                            |
| `queryIntervalMs`   | 默认查询间隔                | `300`                                                                  |
| `uiLayout`          | 界面控件描述                | 见 §5.3 机型与可配置界面                                                        |
| `workModes`         | 工作模式列表及各模式下的显示参数      | `[{"mode":"voltage","params":[...]}, ...]`                             |


### 4A.3 心跳包与状态查询（查询即心跳）


| 子编号   | 需求描述                                                                      | 验收要点                          |
| ----- | ------------------------------------------------------------------------- | ----------------------------- |
| HB-01 | **查询指令作为心跳**：连接成功后，上位机将 Config 中定义的查询指令周期性发送给设备，同时作为心跳检测和数据采集             | 无额外空心跳包开销                     |
| HB-02 | **可配置查询间隔**：每设备独立设置，范围 10ms ~ 1s；DDL 等简单设备默认 1s，VI Sensor 等高频设备可设 10~50ms | 右键设备可修改间隔                     |
| HB-03 | **多指令顺序轮询**：复杂设备（如电源）需发送 3~4 条指令才能获取完整状态；上位机按顺序发送，每条指令等待回复后再发下一条          | 指令列表来自 Config `queryCommands` |
| HB-04 | **超时重试与离线判定**：单条指令未回复 → 重试 N 次（N 可配，默认 3 次）→ 仍无回复 → 设备状态标记为离线             | N 和超时时间均可配置                   |
| HB-05 | **状态机解析**：电源等设备回复无帧头帧尾（纯字符串），解析需基于「上一条发了什么」确定当前回复的解析规则                    | 适配器维护发送状态上下文                  |
| HB-06 | **协议帧头帧尾差异**：A1 型电源有帧头帧尾；AA 型电源纯字符串；匹配器 A 型和自定义协议有帧头帧尾——适配器需分别处理          | Config 中标注 `frameFormat` 类型   |


### 4A.4 连接后 UI 初始化流程

1. 扫描识别 → 确定设备型号 → 加载对应 Config
2. 读取 Config 中的 `uiLayout` → 生成默认界面（滑条数量、模式按钮、参数标签等）
3. 发送首轮查询指令 → 获取设备当前工作模式（如电压模式/功率模式）
4. 根据工作模式 **动态切换界面参数标签和控件数量**（电源在不同模式下显示不同参数）
5. 进入正常心跳轮询，实时更新界面

## 4B. 数据可视化与存储方案（会议纪要 2026-04-22 新增）

### 4B.1 实时波形绘制


| 子编号    | 需求描述                                                                    | 验收要点                      |
| ------ | ----------------------------------------------------------------------- | ------------------------- |
| VIS-01 | **多通道叠加对比**：支持在同一张图表中叠加显示不同设备（如电压、电流、功率）的数据波形，实现多组数据的实时对比               | 可同时选择 N 个数据通道显示（N≥4）      |
| VIS-02 | **字典式数据选择**：所有可绘图的数据项以「字典」形式呈现，用户勾选哪些数据通道参与绘图                           | 字典内容来自已连接设备的 DataPoint 列表 |
| VIS-03 | **采样显示策略**：设备底层数据更新极快（如 5 微秒），上位机显示层采用定时采样（如 100ms 一个点）绘制，未采到的中间值保持上一个点 | 显示采样率可配置；底层数据不因显示降采样而丢失   |
| VIS-04 | **时间戳对齐**：多设备数据绘制在同一时间轴上，通过上位机本地时间戳对齐                                   | 跨设备波形时间轴一致                |
| VIS-05 | **独立绘图窗口**：波形绘图为独立功能窗口，可选择数据通道、设置时间范围、暂停/继续、导出                          | 窗口可独立于主界面操作               |


### 4B.2 数据存储策略


| 子编号      | 需求描述                                                              | 验收要点                    |
| -------- | ----------------------------------------------------------------- | ----------------------- |
| STORE-01 | **先存后画原则**：为防止数据丢失，数据优先写入本地文件或数据库，再根据时间戳读取显示；允许少量显示延迟（毫秒级）        | 异常断电后已存数据完整可恢复          |
| STORE-02 | **异步写入**：数据接收与存储在不同线程；高频数据流（如 VI Sensor）若数据库写入压力过大，自动切换为本地文件直写    | 存储不阻塞数据接收和界面刷新          |
| STORE-03 | **混合存储架构**：数据库存储元数据（文件路径、时间范围、设备信息），实际高频数据存本地文件；低频数据（电源状态等）直接入数据库 | 两种存储路径按数据量自动分流          |
| STORE-04 | **本地 SQLite 数据库**：离线环境设备不联网，所有历史数据存本地 SQLite                      | 支持单文件拷贝到另一台机器还原查看       |
| STORE-05 | **可视化查询界面**：内置历史数据查询面板，支持按时间、设备、参数类型等筛选查询历史记录                     | 百万行查询 ≤ 约定响应时间          |
| STORE-06 | **数据导出**：支持按会话/时间范围导出 CSV / Parquet 格式；导出任务后台执行不阻塞 UI             | 导出文件可被 MATLAB/Python 读取 |
| STORE-07 | **文件拷贝导出**：整个数据库文件可通过 U 盘拷贝到另一台安装同软件的电脑上查看                        | 数据库文件自包含，无外部依赖          |


### 4B.3 VI Sensor 专项数据方案

- VI Sensor 需 **实时动态波形**（非静态图表），数据率极高。
- 建议方案：**环形缓冲区接收 → 异步写本地文件 → 定时采样写入显示缓冲（如 100ms/点） → 绘图引擎渲染**。
- 用户可只观察少量通道（如仅功率），减少绘图压力。
- 多个 Sensor 的数据可在同一窗口对比（叠加同一时间轴）。

## 5. 界面与信息架构

### 5.1 总布局


| 区域   | 内容                                                | 会议补充                                                                             |
| ---- | ------------------------------------------------- | -------------------------------------------------------------------------------- |
| 左侧   | **设备树**；标注接口类型、接口号等                               | 机型下拉、连接方式（串口/网口）                                                                 |
| 中央   | **全部已发现设备**（图标或列表）；**连接状态、简略运行信息**                | RF / 匹配器 / VI / DDL 各状态字段见 SRC-A 概要                                              |
| 详情   | **双击** → 基本信息、全部只读/可写参数（单个/批量）                    | 匹配器：**滑条**、**滚轮**步进；蓝/绿条 **预设 vs 实际**；手动 **实时** 与 **预设后确认** 以 **机型配置 + 设备逻辑** 为准 |
| 右键菜单 | **查询间隔**、**重新握手**、**启停查询**、其他                     | 与会议一致                                                                            |
| 下方   | **终端日志**（可折叠）                                     | —                                                                                |
| 其他   | **日志查询**、**全局状态**、**任务规划与启停**、**数据记录设置**、**系统设置** | 记录项 **任意时刻勾选**                                                                   |


### 5.2 **史密斯圆图**与曲线

- **史密斯圆图**（弹窗待定）：射频电源输出端阻抗；匹配器输入/输出端阻抗；VI Sensor。  
- **曲线**：选数据项；一般参数实时曲线；**快速数据**实时曲线（弹窗待定）。  
- 高频绘图 **降采样显示**，原始数据 **不丢**；波德图等若客户侧解析，上位机提供 **带时间戳原始数据出口**（格式在确认表中选定）。

### 5.3 机型与可配置界面（会议细化）

- **Config 文件驱动 UI 生成**：每个设备 Config 中定义该型号应显示的控件类型（滑条数量、模式按钮、参数标签等），上位机加载 Config 后自动渲染对应界面。
- **工作模式动态切换**：电源等设备有多种工作模式（电压模式、功率模式等），切换模式后界面的参数标签、控件数量、显示内容随之动态变化。例如电压模式下显示电压相关 2~3 个参数，功率模式下参数变为 4 个。
- **匹配器电机数量可变**：不同型号匹配器电机数量不同（2 个/4 个），界面滑条数量根据 Config 中 `motorCount` 字段动态生成。
- **主界面卡片信息可配**：每个设备在中央总览中显示的「小框子」内参数由 Config 定义，不同型号显示不同变量。
- 与握手自描述合并：**数据模型驱动 UI**。

### 5.4 设备详情页进入流程（会议确认）

1. 识别阶段已确定设备型号 → 加载对应 Config 默认界面
2. 连接后首轮查询获取设备当前状态（工作模式、参数值等）
3. 同步状态到界面控件（如滑条位置、模式选择、开关状态）
4. 后续通过心跳查询持续更新界面数据

## 6. 任务计划

### 6.1 功能描述

- **按时间线顺序**执行（**非**日历定时）；**多设备、多指令** 编排；**所有可下发指令** 及上位机操作（如 **启动记录**）为步骤；**重复 N 次**、循环；**毫秒级** 延时与精度。

### 6.2 指令字典库（会议新增）

- **设备关联指令库**：任务计划中可添加的指令来源于已连接设备实际支持的指令集（从 Config 和握手字典中获取），而非用户手动输入。
- **字典式选择 UI**：新建任务步骤时，用户从预设指令库中**下拉选择**目标设备和指令，避免输入错误。
- **动态更新**：指令库随设备连接/断开动态更新；设备未连接时其指令不可选（灰显提示）。

### 6.3 执行规则（「任务计划执行规则」）

1. 自启动起顺序执行。
2. 执行指令 → 延时 → 下一指令。
3. 支持循环。
4. **无需对齐系统时间点**。
5. 循环间隔由任务内 **毫秒延时** 控制。
6. 支持组合动作：启动记录、设置模式、开关、延时等。
7. 所有指令可配置为任务步骤。

### 6.4 与数据记录的衔接

- 任务中「启动/停止数据记录」与 **记录项配置** 共用 **设备—参数树** 元数据。
- 数据记录需包含 **时间戳**，并按时间线顺序执行，支持快速记录和缓存到本地。

## 7. 数据记录

### 7.1 一般数据收集

- 每条记录 **带时间戳**；指定 **某设备某参数**；来源：**旧设备已知表** + **新设备握手上报域**；**加载/保存记录配置**。  
- 软件打开后任意时刻可勾选；未连接设备时可记录 **本地缓存/状态**（无更新则保持）。

### 7.2 快速数据收集

- **1ms 一条**、**UDP** 入本地。  
- 远期 **连续推送/大带宽** 与查询式 **解耦**；本期以快速通道做 **原型验证**。

### 7.3 存储形态（会议细化）

- 原型：**本地文件**（高频数据优先文件直写，避免数据库写入瓶颈）。
- 演进：**SQLite** + **设备表 + 日志表 + 会话表**。
- **离线优先架构**：设备通常不联网，所有数据存本地 SQLite；支持通过 **文件拷贝** 方式（U 盘等）导出到另一台机器进行离线分析。
- **可视化查询界面**：内置历史数据浏览器，支持按时间范围、设备、参数类型等条件查询和筛选。
- **远期扩展**：量产后考虑定时推送数据到云端数据库（需联网环境），用于批量失效分析与数据统计（三期规划）。
- 详细设计需 **表结构草案、滚动删除策略、磁盘水位监控、导出格式**。

## 8. 交互规则

- 上位机 **定时查询** 并 **实时刷新**；状态、数值、报警 **实时展示**；**在线设置** 弹窗；**双击放大**；**任务计划** 启停控制。

## 11.2 功能规格细化（按模块，研发可拆 Epic/Story）

> 下列为 **规格级** 描述：若与签字 ICD 冲突，以 ICD 为准；表中「待 ICD 填」项由甲方在协议包中补全。

### 11.2.1 设备抽象与运行时模型（全局）


| 概念                         | 定义                                            | 行为要求                                                                        |
| -------------------------- | --------------------------------------------- | --------------------------------------------------------------------------- |
| **设备实例 DeviceInstance**    | 一次「逻辑设备」绑定：机型 + 通讯配置 + 协议适配器                  | 唯一 `deviceId`；支持启用/禁用；禁用后不参与轮询与任务                                           |
| **连接配置 ConnectionProfile** | 串口（COM、波特率、校验等）或网络（IP、端口、TCP 角色）或 EC（从站地址/别名） | 配置变更后需 **显式重连** 或向导式「应用并连接」；非法组合禁止保存（校验提示）                                  |
| **协议适配 ProtocolAdapter**   | 旧固定协议 / 新握手协议 / Modbus / 自定义帧 等插件             | 同一进程内可多实例；崩溃隔离：**单适配器异常不得拖垮整进程**（恢复策略见下）                                    |
| **数据点 DataPoint**          | 可读或可写最小单元，带元数据                                | 元数据至少含：`pointId`、显示名、类型（数值/枚举/字符串/布尔/字节块）、单位、小数位、`rw`、分组路径、`alarmClass`（若有） |
| **轮询槽 PollingSlot**        | 绑定在 DeviceInstance 上的周期调度单元                   | 周期可配；可暂停；暂停时 UI 显示「冻结」态；恢复后继续                                               |


**连接状态机（建议实现，便于 UI 与日志统一）**


| 状态             | 含义              | 进入条件           | 离开/下一状态                                           |
| -------------- | --------------- | -------------- | ------------------------------------------------- |
| `Scanning`     | 扫描识别中（会议新增）     | 用户点击「扫描设备」     | 识别成功 → `Connecting`；无回复 → `NotFound`              |
| `NotFound`     | 扫描未发现（会议新增）     | 所有 Config 均无匹配 | 用户重新扫描 → `Scanning`                               |
| `Disconnected` | 未连接             | 初始/用户断开        | 用户发起连接 → `Connecting`                             |
| `Connecting`   | 物理链路建立中         | 发起连接           | 成功 → `Handshaking` 或 `Ready`（旧协议无握手）；失败 → `Fault` |
| `Handshaking`  | 新协议握手中          | 新设备            | 成功 → `Ready`；超时/校验失败 → `Fault`                    |
| `Ready`        | 可正常查询/设置        | 握手完成或旧协议就绪     | 通讯异常 → `Degraded` 或 `Fault`                       |
| `Querying`     | 本轮查询进行中（可选显式态）  | 调度器触发          | 结束回 `Ready`                                       |
| `Degraded`     | 降级（如仅读不可写、偶发超时） | 连续 N 次超时（N 可配） | 恢复 → `Ready`；持续失败 → `Fault`                       |
| `Fault`        | 故障需人工干预         | 严重错误           | 用户「重连/重新握手」→ `Connecting`                         |


**单适配器异常恢复（Must 级工程要求）**

- 连续超时、解析异常、非法帧：写入 **设备故障日志** + 中央总览 **红色状态**；达到阈值后可 **自动暂停查询**（可配）以免洪泛。  
- **不得**因单设备死锁导致全局 UI 阻塞（通讯线程与 UI 线程解耦，SRC-C）。

## 14. 测试与验收（闭环）

### 14.1 场景用例（抽样）

1. **自动扫描**：点击「扫描设备」→ 进度条推进 → 3 台设备识别成功 → 设备树自动添加 → 各设备在线状态正确。
2. **Config 驱动 UI**：匹配器识别后 → 自动生成 2 个电机滑条 → 切换为 4 电机型号 → 滑条数量自动变为 4。
3. **心跳查询**：连接电源 → 周期性发送 HPV 指令 → 界面实时刷新 → 拔网线 → 3 次重试后标记离线。
4. **工作模式切换**：电源处于电压模式 → 切换为功率模式 → 界面参数标签和控件数量动态变化。
5. **连接**：新增串口对象 → 选专用协议 → 连 RF A → 中央总览字段刷新正确。
6. **握手**：新设备上电 → 发握手 → UI 树与 ICD 一致 → 单项/全量读写成功。
7. **匹配器**：拖动滑条 → 与 **参考 Qt 程序** 同机对比一致。
8. **实时波形**：选择 3 个数据通道 → 同一时间轴叠加显示 → 暂停/继续 → 数据不丢失。
9. **记录**：勾选 3 参数 → 运行 1h → 导出 CSV/SQLite → 时间戳连续。
10. **数据存储**：VI Sensor 高频数据 → 异步写本地文件 → 数据库记录元数据 → 断电后数据完整。
11. **任务**：从指令字典选择步骤 → 循环 10 次「开 RF → 延时 →关 RF」→ 中止 → 审计日志完整。
12. **离线数据导出**：SQLite 文件拷贝到另一台机器 → 软件打开 → 可查看历史数据。
13. **快速数据**：1ms UDP 注入 → 丢包率 < 约定阈值。

### 14.2 通过准则

- §11 所有 **Must** 100%；**Should** ≥90% 或签署豁免；**Could** 不阻断发布。
