# 马术教培小程序 — 功能方案 v1.0

## 模块一：产品概述与目标

### 一句话产品定位

面向马术俱乐部家长、学员、教练与门店管理员的微信小程序与管理后台，解决马术课程售卖、线下核销、自动约课、课包扣课、赛事活动传播和装备售卖的一体化管理问题。

### 核心价值主张

| 价值点 | 说明 |
|---|---|
| 课程交易线上化 | 家长可以在线浏览体验课、考级课、正式课、大师课，并完成下单支付。 |
| 教务履约闭环 | 家长预约课程后，系统自动确认；到店先核销，教练后扣课，课时变动实时通知家长。 |
| 运营传播增强 | 竞赛获奖、开业活动、充值活动可推送给客户，竞赛喜报可分享给其他家长或微信用户。 |
| 周边商品变现 | 头盔等马术装备支持在线展示、下单和订单管理。 |

## 模块二：目标用户画像

### 1. 家长用户

| 维度 | 描述 |
|---|---|
| 人口特征 | 以儿童、青少年学员家长为主，重视安全、师资、课程成果和时间便利性。 |
| 核心诉求 | 想快速了解课程、购买课包、预约合适教练和时间，并随时查看剩余课时。 |
| 使用场景 | 课前查看课程与活动、购买课程或装备、预约上课、接收课前提醒和扣课通知、分享孩子获奖喜报。 |

### 2. 教练用户

| 维度 | 描述 |
|---|---|
| 人口特征 | 马术俱乐部教练，关注自己当日课表、上课学员和课后扣课记录。 |
| 核心诉求 | 快速查看预约学员，在核销完成后准确扣课，减少纸质登记和口头确认。 |
| 使用场景 | 每天上课前查看课表；上课后对已核销预约执行点名扣课。 |

### 3. 管理员 / 运营人员

| 维度 | 描述 |
|---|---|
| 人口特征 | 俱乐部负责人、前台、教务或运营人员。 |
| 核心诉求 | 管理课程、装备、教练档期、线下课包、活动推送和核销订单。 |
| 使用场景 | 上架课程与装备、录入线下签约课包、配置教练可约时段、发布竞赛喜报和活动消息。 |

## 模块五：全部业务流程

### 1. 课程购买与到店核销流程

1. 家长在小程序首页进入课程分类页。
2. 家长选择体验课、考级课、正式课或大师课。
3. 系统展示对应课程列表。
4. 家长进入课程详情页，查看课程说明、价格、课时和购买须知。
5. 家长提交订单并通过微信支付完成付款。
6. 系统生成订单，并在订单详情页展示门店地址和核销码。
7. 家长到店后向前台出示核销码。
8. 前台扫码核销。
9. 系统记录核销时间、核销人和核销状态。
10. 若订单为单次课程，核销后履约结束；若订单转为课包，则进入后续约课和扣课流程。

### 2. 课包购买、预约、核销、扣课流程

1. 家长通过线上购买课包，或管理员把线下签约课包录入后台。
2. 家长进入约课页，选择学员、课型、教练、日期和时间段。
3. 系统自动校验课包余额、教练档期、场地容量和学员时间冲突。
4. 校验通过后，系统自动生成已确认预约。
5. 系统在上课前一天向家长发送课前提醒。
6. 家长到店后，前台先完成本节预约的到店核销。
7. 教练在教练端查看已核销课程。
8. 教练确认学员已上课后执行扣课。
9. 系统生成扣课流水，扣减课包剩余课时。
10. 系统向家长推送本次扣课和剩余课时。

### 3. 装备购买流程

1. 家长进入装备商城页。
2. 家长选择头盔等装备商品。
3. 系统展示商品规格、价格和库存。
4. 家长提交装备订单并完成支付。
5. 系统生成装备订单并扣减库存。
6. 后台处理发货或自提，具体履约方式待确认。

### 4. 竞赛喜报发布与分享流程

1. 管理员在后台新增竞赛喜报。
2. 管理员填写获奖内容、图片、推送受众和分享卡片信息。
3. 系统发布喜报并在小程序展示。
4. 系统可向配置的受众发送订阅消息。
5. 家长打开喜报详情页。
6. 家长点击分享，把喜报分享给其他家长或微信用户。
7. 接收人点击分享卡片后进入喜报详情页。

## 模块八：技术实现难点提示

| 难点 | 难点所在 | 潜在风险 | 建议应对思路 |
|---|---|---|---|
| 自动约课冲突校验 | 系统需要同时校验教练档期、场地容量、学员时间冲突和课包余额。 | 多人同时抢同一时段可能导致超约。 | 预约提交时使用实时余位校验，并在确认时锁定名额。 |
| 预约、核销、扣课顺序一致性 | 业务要求必须先预约、再核销、后扣课。 | 教练误扣课或重复扣课会导致课时争议。 | 扣课按钮仅在预约已核销且未扣课时展示；扣课流水强制绑定预约和核销记录。 |
| 线下课包录入与审计 | 管理员可录入线下课包，涉及课时资产。 | 误录、私自调账或家长争议。 | 课包录入、修改、冲正必须记录操作日志；普通账号不能删除日志。 |
| 微信订阅消息限制 | 小程序订阅消息需要用户授权，且模板能力受微信规则限制。 | 消息无法稳定触达所有家长。 | 关键节点引导家长订阅；后台保留站内消息列表作为补充。 |
| 竞赛喜报分享合规 | 喜报可能涉及未成年人照片、姓名和成绩。 | 不当展示可能引发隐私和肖像权风险。 | 默认最小必要展示；后台发布前提示运营确认授权材料。 |
| 装备商城履约方式未定 | 装备可能到店自提，也可能快递发货。 | 若规则未定，订单状态和售后流程无法完整设计。 | 首期建议先确认到店自提或快递之一；另一种作为二期扩展。 |
| 考级课业务细节未定 | 考级课可能涉及级别、报名资格、准考证、成绩等。 | 若客户后续要求考级全流程，范围会扩大。 | 首期先按课程售卖和详情展示处理；考级报名与成绩查询列为后续扩展。 |
| 会员卡 / 充值余额未定 | 访谈提到充值活动，但未确认余额支付。 | 如果加入余额支付，会影响支付、退款和财务对账。 | 首期先做充值活动内容展示；余额账户另行确认后再纳入范围。 |
