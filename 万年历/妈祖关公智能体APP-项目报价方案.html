妈祖 / 关公 神仙智能体 · 双 APP 项目报价方案

基于《产品定位与功能需求说明》整理，需求原文不作增删改，仅做结构化呈现与商务报价拆解。

项目形态：妈祖智能体 APP、关公智能体 APP（双端独立运营，视觉与 IP 区隔） 报价币种：人民币（含税与否以合同约定为准） 总价：¥189,200

一、需求范围总览（与需求文档一致）

  1. 产品定位与核心形态：双 APP；统一视觉体系；NFC 一碰拉起 → 专属落地页 → 自动识别妈祖 / 关公下载链路。
  2. 欢迎页（新用户首屏）：IP 主视觉与动效；主副标题；三列卖点；琥珀金主 CTA；登录入口与注册引导文案。
  3. 登录注册（统一 4 步）：通用头部（返回、头像、进度条、步骤标注）；Step1 国内手机号 / 国外邮箱、验证码与密码登录、协议勾选；Step2 昵称性别、阳历阴历联动、出生信息、闰月、时辰（含「不清楚」）、城市、实时「属相 / 日主」反馈；Step3 流程占位；Step4 兴趣多选。
  4. 基础通用：页面模糊加载过渡；每日 / 每周推送（黄历解读、运势周报）。
  5. 首页：新用户助运壁纸；顶部信息栏；日历主卡片；今日运程核心区；快捷入口；运势详情（雷达图、解读、幸运时段、助运元素、个性化建议）。
  6. 底部导航：首页、择日、AI 陪伴、我的。
  7. 私属黄历解读：日级缓存；客户 Java 黄历算法；四柱八字等命盘；真太阳时；向量库存储；天人合一评分与宜忌、吉时、助运元素、大模型文案生成。
  8. 妈祖 / 关公择日：中心 IP + 环绕毛玻璃分区旋转、触摸放大；事件描述 → 时间范围 → 吉日结果（类型、匹配星级、备选 ≥3、记录与报告动效）。
  9. AI 私属陪伴：文字 / 语音（转写）、图片识别上传、开场白、快捷问题、主动互动、历史记录。
  10. 助运推荐：节气 / 助运壁纸生成、说明、后台替换、下载相册、分享水印。
  11. IP 文化内容：顶部循环动画（后台可换）、文化介绍（含妈祖海洋文化、关公洛阳文旅等）、分享水印。
  12. 个人中心：资料展示与修改边界、密码；VIP 体系与权益差异；签到与各类记录；反馈与关于页。
  13. 多语言与合规：国内中文 / 国外英文；注册前强制隐私免责；全站底部统一免责声明（娱乐参考、非专业建议）。

二、实施节奏（与贵方口径一致）

  • UI 设计阶段：约 10–12 个工作日（单价 ¥1,000 / 人天）。
  • 开发阶段：UI 定稿后进入联调开发；前端 ¥1,200 / 人天，后端（Java） ¥1,500 / 人天。

总价与人天闭合说明：评估周期为 10–12 人天。在「仅含 UI + 前端 + 后端」且单价固定为 ¥1,000 / ¥1,200 / ¥1,500、总价锁定 ¥189,200 时，10 人天、12 人天的 UI 费用与剩余开发预算无法同时拆成整数前端人天 + 整数后端人天；区间内11 人天可与前后端整人天精确闭合。故本报价表按 11 人天 UI（¥11,000）示例入账。若实际执行为 10 或 12 人天且仍锁总价，建议在合同中采用打包人天或对前后端人天做合同内微调（不必对外显式拆分为非整数人天）；若按实做 UI 人天结算，则 10 / 12 人天与表内开发人天组合需另行对齐总价。

四、交付边界提示（不改变需求，仅商务说明）

  • 第三方费用（应用商店、短信 / 邮件、对象存储、CDN、推送通道、大模型 API、语音与图片识别等）通常不含在人天单价内，或由甲方实报实销。
  • 客户提供的 Java 黄历算法以源码 / SDK 形式接入为前提；若需反编译或重写，需另评估。
  • 「后台可配置」项需明确后台形态（独立管理端或配置中心）；若未包含在本报价的人天内，应单列。
