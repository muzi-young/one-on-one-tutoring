PART 01

产品概述

妈祖智能体 APP 与关公智能体 APP 双端独立运营：整体视觉配色、品牌形象、IP 风格保持统一，仅核心 IP 形象与文化内容区分。用户端支持 NFC 一碰拉起，自动识别并跳转对应 APP 下载落地页；落地页自动判断展示妈祖 / 关公形象与下载入口。

项目总报价（见报价章）

产品定位与核心价值（与需求文档一致）

  • 产品定位：面向大众用户的「神仙智能体」私属万年历与生活陪伴应用，融合传统黄历、个人命盘、择吉、AI 对话与助运内容。
  • 核心入口：NFC 触碰 → 打开专属下载落地页 → 自动识别妈祖 / 关公分支与下载入口。
  • 新用户路径：欢迎页（IP 主视觉、动效、三列卖点、琥珀金主 CTA）→ 统一 4 步注册 / 登录（国内手机号 / 国外邮箱等多方式）→ 资料与兴趣完善。
  • 首页价值：农历生辰摘要、当日皇历与运程、宜忌速览、快捷入口（择天时 / 窥天机 / 助运势 / 万事通）及运势详情（五维雷达、助运元素、个性化建议）。
  • 私属黄历：客户 Java 黄历算法 + 四柱八字等命盘；天人合一综合评分；宜忌与吉时、助运元素；算法字段结合大模型生成解读与建议。
  • 合规表达：注册前强制展示《用户隐私保护及免责须知》；全页底部统一标注「基于经典国学算法推演建议 · 仅作生活娱乐参考」，不构成医疗、法律、投资等专业建议。

PART 03

技术架构方案

移动端双 APP 采用 uni-app 一套代码多端发行（妈祖 / 关公独立应用标识与主题资源分包）+ Java 后端服务；黄历与命理计算采用客户提供的 Java 算法；AI 解读与对话通过大模型与向量库编排实现。

客户端层 · Apps（uni-app）
uni-app + Vue 3（双 APP 独立打包配置） 妈祖 APP（独立包名 / AppID / 品牌资源） 关公 APP（独立包名 / AppID / 品牌资源） 欢迎页 / 注册四步 / 首页运程 择日动效页 / AI 会话 / 个人中心 NFC 落地 H5（uni-app 条件编译或独立页）
接入层 · Gateway
HTTPS API 网关 鉴权与刷新策略（可配置会话时长） 静态资源与壁纸 CDN
服务层 · Java 业务中台
用户与多方式登录（手机 / 邮箱 / 密码 / 验证码） 资料与兴趣标签 客户 Java 黄历算法接入 命盘计算 · 真太阳时 · 天人合一评分 择日引擎与记录 / 报告 会员与限次策略（后台可配） 推送任务（每日黄历 / 每周周报） 向量库与个人命盘存储 大模型编排（运程文案、对话、图片理解） 壁纸生成与分享水印参数
数据层 · Data
关系型数据库（用户、订单、记录） 缓存（日运、黄历日粒度缓存） 对象存储（头像、壁纸、上传图片）
第三方与合规 · Integrations
短信 / 邮件验证码 应用内推送 / 服务号模板（按实际通道） 大模型与多模态 API 语音转写与图片识别服务 应用商店分发与 NFC URL

架构设计要点

  • 双 APP 工程策略（uni-app）：共享业务页面与网络层，通过 manifest 多环境、主题 SCSS 变量、static 资源分包与条件编译区分妈祖 / 关公 IP 与文化模块，分别打包上架 App Store / 各安卓应用市场。
  • Java 算法边界：皇历、四柱、纳音、十神、择日核心数值均在服务端 Java 内完成，保证与天文台闰年闰月及客户算法一致；客户端负责展示与动效。
  • 缓存策略：私属黄历解读按「自然日 + 用户」维度缓存，满足「一天首次生成、二次秒开」体验。
  • AI 层：结构化算法输出字段作为 Prompt 输入，生成每日一言、深度解读、穿搭建议等自然语言内容。
  • 合规：注册前强制协议页、全站底部免责文案由配置中心或前端统一组件承载，便于多语言切换。

PART 05

实施排期计划

先完成 UI 设计（约 10~18 个工作日，视评审轮次而定），UI 定稿后进入开发；开发按阶段并行推进。人天规划口径可参考 ¥1,000 / UI 人天、¥1,200 / 前端人天、¥1,500 / Java 人天，与 PART 06 商务拆分合计相互对照；日历周数以团队配置为准。

Phase 0
UI / UX 设计（开发前置）
约 10~18 个工作日（与评审轮次相关）
  • 双 APP 统一设计系统 + 妈祖 / 关公 IP 差异（色彩、组件、插画位）
  • 欢迎页、注册四步、首页、运程详情、择日、AI 会话、个人中心、文化页等核心界面高保真
  • 关键动效说明（光晕脉冲、琥珀金 CTA、择日环绕旋转、加载模糊过渡）
  • 切图与标注交付开发
Phase 1
基础工程 + 账号与注册链路
开发第 1 阶段（并行）
  • uni-app 双 APP 工程骨架（双 manifest / 环境变量）、主题切换、网络层与多语言（vue-i18n）
  • Java：用户体系、手机 / 邮箱验证码、密码登录、会话策略、协议记录
  • 客户端：欢迎页、Step1~4 流程、进度头与表单校验
Phase 2
首页 · 黄历 · 运程 · 推送
开发第 2 阶段
  • Java：接入客户黄历算法、命盘计算、天人合一评分字段、日缓存 API
  • 大模型编排：每日解读、宜忌说明、幸运密码与穿搭建议生成链路
  • uni-app 客户端：首页信息栏、日历主卡片、今日运程区、运势详情雷达与文案展示
  • 每日 / 每周推送对接与后台配置项
Phase 3
择日 · AI 陪伴 · 助运壁纸
开发第 3 阶段
  • 择日：事件模型、时间范围检索、结果排序与记录 / 报告接口
  • uni-app：择日动效页、结果列表与详情、记录列表动效
  • AI：会话、语音转写、图片上传识别、历史记录
  • 壁纸生成、下载、分享水印；新用户壁纸弹窗
Phase 4
会员 · 我的 · 文化模块 · NFC 落地
开发第 4 阶段
  • VIP 订阅与限次策略（对话、择日、壁纸、运程提前查看等后台可配）
  • 个人中心、签到与各类记录、反馈与关于页
  • IP 文化动画与图文内容后台可替换
  • NFC 落地页识别与商店跳转联调
Phase 5
联调测试 · 上架交付
开发第 5 阶段
  • 双 APP 端到端回归、性能与安全基线检查
  • 合规文案与隐私流程走查
  • 应用商店资料与生产环境部署、验收交付
